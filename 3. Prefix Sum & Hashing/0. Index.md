## Variants of Prefix Sum & Hashing (cluster the problems)

| Variant | Problems | Signature |
|---------|----------|-----------|
| **Complement lookup** (value-only, no prefix sum) | 1 | hash map `value → index`; for each new element ask "is `target - x` already in the map?"; **check before insert** so an element never pairs with itself |
| **Static range aggregate** (prefix sum, no hashing) | 303 | precompute `ps` of size `n+1` with `ps[0]=0`; answer `sumRange(l,r) = ps[r+1] - ps[l]` in O(1). Reach for it whenever the array is immutable and queries are many |
| **2D static range aggregate** (2D prefix sum + inclusion–exclusion) | 304 | precompute `ps[i][j]` of size `(m+1)×(n+1)` with a zero border; both build and query use `+ − − +` over four top-left rectangles. Lift of 303 into a second dimension |
| **Counting subarrays by prefix value** (prefix sum + frequency map) | 560 | running `sum`; `count += cnt[sum − k]` then `cnt[sum]++`. Seed `cnt[0] = 1`; **lookup before insert** so an element can't self-pair when `k == 0`. Reach for it whenever the question says "count subarrays with sum = / divisible by / equal to a property" and negatives are possible |
| **Counting subarrays by prefix mod K** (prefix mod + frequency array) | 974 | running `sum`; `rem = ((sum%k)+k)%k`; `count += freq[rem]++`. Seed `freq[0]=1`. Prefer `vector<int>(k)` over `unordered_map` — remainders are dense in `[0,k-1]`. The modular cousin of 560 |
| **Exists / longest subarray by prefix mod K** (prefix mod + earliest-index map) | 523 | running `sum`; `rem = sum%k`; store `rem → earliest index` (**not** a frequency); on a repeat, valid iff `i − firstIdx ≥ 2`. Seed `{0:-1}`, **insert only on first sight**. Fork from 974: *index for exists/longest, frequency for count*. `k` can be huge ⇒ `unordered_map` |
| **Longest subarray summing to a target** (recode → prefix balance + earliest-index map) | 525 | recode the alphabet so the target property becomes "**prefix sums are equal**" (here `0→−1, 1→+1` ⇒ equal 0s/1s = balance repeats); store `balance → earliest index`, seed `{0:-1}`, `res = max(res, i − firstIdx)`. Generalizes to "longest subarray with sum k" (`cnt[sum−k]`). Longest cousin of 523 |

## When to reach for Prefix Sum & Hashing
1. "Two elements / subarray with property X" where the property decomposes into **"have I already seen Y?"** — Y is a value (Two Sum), a running sum (Subarray Sum Equals K), a sum mod K, a parity / count delta, etc.
2. "**Count** the number of subarrays with sum = K / sum divisible by K / equal 0s and 1s" — running prefix + frequency map of past prefix values.
3. **Range sum queries** on an immutable array — precompute prefix sums for O(1) per query.
4. Need an O(1) lookup of "complement" or "earlier state" to drop a brute-force O(n²) to O(n).

## Common Pitfalls Across the Pattern
- **Check before insert.** Always query the map *before* recording the current element, otherwise a value can pair with itself (`[3,3]` in Two Sum, `nums[i]==0` in Subarray Sum if `k==0`, etc.).
- **Seed the prefix map with `{0: 1}`** when counting subarrays — a prefix sum that already equals `k` corresponds to a subarray starting at index 0, which needs the empty-prefix sentinel.
- **`find()` over `count()` + `[]`.** `count` + `[]` does two hash lookups; `find()` returns the iterator in one. Idiomatic and faster.
- **Overflow.** Running prefix sums on `10^4` elements of value `10^7` exceed `int`; reach for `long long` whenever the constraints allow it.
- **Don't sort.** Sorting destroys indices and breaks the contiguous-subarray invariant — this is what distinguishes the family from Two Pointers on sorted input.
- **Normalize C++ negative modulo.** `%` returns a remainder with the dividend's sign, so `-7 % 5 == -2`. Always fold with `((x % k) + k) % k` before using `rem` as a key — otherwise equivalent prefixes land in different buckets.
- **Use `vector` over `unordered_map` for dense bounded keys.** Indices for prefix sums (303) and remainders mod K (974) are dense in a known range; a flat `vector` skips hashing entirely and is faster + clearer.

## C++ Prefix Sum & Hashing templates
```cpp
// Complement lookup — Two Sum (LC 1)
vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int, int> mpp;                       // value -> index
    for (int i = 0; i < (int)nums.size(); i++) {
        int req = target - nums[i];
        auto it = mpp.find(req);
        if (it != mpp.end()) return {it->second, i};   // complement seen earlier
        mpp[nums[i]] = i;                              // record AFTER lookup
    }
    return {};
}

// Count subarrays with sum == k — Subarray Sum Equals K (LC 560)
int subarraySum(vector<int>& nums, int k) {
    unordered_map<int, int> cnt{{0, 1}};               // empty-prefix sentinel
    int sum = 0, ans = 0;
    for (int x : nums) {
        sum += x;
        ans += cnt[sum - k];                           // # earlier prefixes giving target window
        cnt[sum]++;
    }
    return ans;
}

// Range sum query on immutable array — LC 303
struct NumArray {
    vector<int> ps;                                    // ps[i] = nums[0..i-1]
    NumArray(vector<int>& nums): ps(nums.size() + 1, 0) {
        for (int i = 0; i < (int)nums.size(); i++) ps[i+1] = ps[i] + nums[i];
    }
    int sumRange(int l, int r) { return ps[r+1] - ps[l]; }
};

// Count subarrays with sum divisible by k — Subarray Sums Divisible by K (LC 974)
int subarraysDivByK(vector<int>& nums, int k) {
    vector<int> freq(k, 0);                            // remainder -> count
    freq[0] = 1;                                       // empty-prefix sentinel
    int sum = 0, count = 0;
    for (int x : nums) {
        sum += x;
        int rem = ((sum % k) + k) % k;                 // normalize negatives
        count += freq[rem]++;                          // read old, then bump
    }
    return count;
}

// 2D range sum on immutable matrix — LC 304 (+ − − + on four top-left rectangles)
struct NumMatrix {
    vector<vector<int>> ps;                            // ps[i][j] = sum of (0..i-1, 0..j-1)
    NumMatrix(vector<vector<int>>& a) {
        int m = a.size(), n = a[0].size();
        ps.assign(m + 1, vector<int>(n + 1, 0));
        for (int i = 1; i <= m; i++)
            for (int j = 1; j <= n; j++)
                ps[i][j] = a[i-1][j-1] + ps[i-1][j] + ps[i][j-1] - ps[i-1][j-1];
    }
    int sumRegion(int r1, int c1, int r2, int c2) {
        return ps[r2+1][c2+1] - ps[r1][c2+1] - ps[r2+1][c1] + ps[r1][c1];
    }
};
```
