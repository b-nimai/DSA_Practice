# 9. Maximum Number of Vowels in a Substring of Given Length (LC 1456) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/maximum-number-of-vowels-in-a-substring-of-given-length/)

**Pattern:** Sliding Window — **fixed size** (window length given = `k`), maximize a count over the window
**Trigger:** "any substring of **length k**" + "maximum number of …" → strip the story: **fixed window of width `k`, track one running count, take the max**. Slide one char in (`right`), one char out (`left`).

## Problem (1-liner)
Given string `s` and int `k`, return the **max number of vowels** (`a e i o u`) in any length-`k` substring of `s`.
**Example:** `s="abciiidef", k=3` → `3` (`"iii"`). `s="aeiou", k=2` → `2`. `s="leetcode", k=3` → `2`.

## Idea (1-liner)
"length k" is a dead giveaway for a **fixed-size** window — same family as [1. Maximum Average Subarray I](1.%20Maximum%20Average%20Subarray%20I.md) and the anagram problems [5](5.%20Permutation%20in%20String.md) / [6](6.%20Find%20All%20Anagrams%20in%20a%20String.md). Keep a running `vowels` count; once the window reaches width `k`, record `max`, then drop the leftmost char's contribution and advance. No map needed — the only state is a single integer.

## Approach
1. `maxVowel = 0`, `vowels = 0`, `left = 0`.
2. For each `right`: if `s[right]` is a vowel, `vowels++`.
3. If window not yet width `k` (`right - left + 1 < k`), keep growing.
4. Else: `maxVowel = max(maxVowel, vowels)`; if `s[left]` is a vowel, `vowels--`; `left++`.
5. Return `maxVowel`.

## Code (clean canonical — fixed window, O(n)/O(1))
```cpp
int maxVowels(string s, int k) {
    auto isVowel = [](char c) {
        return c == 'a' || c == 'e' || c == 'i' || c == 'o' || c == 'u';
    };
    int vowels = 0;
    for (int i = 0; i < k; i++)                 // build the first window
        if (isVowel(s[i])) vowels++;
    int best = vowels;
    for (int right = k; right < (int)s.size(); right++) {
        if (isVowel(s[right])) vowels++;        // slide one in
        if (isVowel(s[right - k])) vowels--;    // slide one out
        best = max(best, vowels);
        if (best == k) return k;                // early exit: can't beat k
    }
    return best;
}
```
**TC:** O(n)  **SC:** O(1) — single int of state. Same shape as the #1 "build first `k`, then `+a[i] -a[i-k]`" template.

## My Solution
```cpp
class Solution {
public:
    int maxVowels(string s, int k) {
      int maxVowel = 0, vowels = 0;
      int left = 0;

      for (int right = 0; right < s.size(); right++) {
        if(s[right] == 'a' || s[right] == 'e' || s[right] == 'i' ||
        s[right] == 'o' || s[right] == 'u') vowels++;

        if(right - left + 1 < k) continue;
        maxVowel = max(maxVowel, vowels);
        if(s[left] == 'a' || s[left] == 'e' || s[left] == 'i' ||
        s[left] == 'o' || s[left] == 'u') vowels--;
        left++;
      }
      return maxVowel;
    }
};
```
✅ Correct and accepted. O(n) / O(1) — already optimal in complexity. The `left/right` form with `if (right-left+1 < k) continue;` is a perfectly valid way to express a fixed window: grow until width `k`, then for every later step record-then-evict-then-advance so width stays pinned at `k`. State is a single `int` — no map, which is exactly right for "count one property over a fixed window."

## Key Insight — fixed window = one in, one out, width never changes
Because `k` is fixed, you never *search* for the right width (that's the variable-window job in #3/#4/#7/#8). Each step past the warm-up does the **same constant work**: `+s[right]`, `-s[left]`, advance `left`. The window is a rigid frame sliding along the string. Recognising "length k" → fixed frame is the whole trick; the rest is bookkeeping one counter.

| | My form (`left/right` + `continue`) | Canonical (`build k`, then `i`/`i-k`) |
|---|---|---|
| Warm-up | implicit via `continue` until width `k` | explicit first loop builds first window |
| Evict index | `s[left]`, then `left++` | `s[right - k]` (no `left` var) |
| Complexity | O(n) / O(1) | O(n) / O(1) |
| Notes | one loop, slightly more state (`left`) | two loops, no `left`; trivial early-exit |

## Dry run — `s="leetcode", k=3` (my form)
| right | s[right] | vowels after add | width = r-l+1 | width < k? | maxVowel | evict s[left] | left |

## Other Approaches
- **Build-first-`k` then slide (`i` / `i-k`)** — the canonical form above; drops the `left` variable and reads as the exact #1 template. Best to *state first* in an interview since it makes the fixed-window structure obvious.
- **Prefix-sum of vowels** — `pre[i]` = #vowels in `s[0..i)`; answer = `max(pre[i+k] - pre[i])`. Also O(n)/O(n); only worth it if you need many different `k` queries — overkill for a single `k`.

## Refinements still applicable
- **Deduplicate the vowel test** — the 5-way `||` chain is written twice; a typo in one copy silently corrupts the answer. Extract an `isVowel` lambda (`auto isVowel = [](char c){ return c=='a'||...; };`) and call it for both the add and the evict.
- **Early exit `if (maxVowel == k) return k;`** — a width-`k` window can hold at most `k` vowels; once you hit `k` there is nothing better to find. Turns inputs like `"aeiou…"` into an instant return. Pure win, worst case unchanged.
- **Signed/unsigned** — `right < s.size()` compares `int` with `size_t`. Cache `int n = s.size();` or cast `(int)s.size()` to silence the warning (harmless here given `s.length ≤ 1e5`).

## Pitfalls
- **"length k" → fixed window, not variable.** Don't grow/shrink searching for a width — the width *is* `k`. Mixing in a `while`-shrink (the #3/#4/#7/#8 template) is the classic wrong-template bug here.
- **Evict in lockstep with insert.** Every step at full width must remove exactly one char (`s[left]` / `s[right-k]`); forget the decrement and `vowels` only ever grows → wrong answer.
- **Record at the right moment** — take `max` only when the window is exactly width `k` (after warm-up), not during the build-up.
- Same engine as the fixed-window problems **#1 / #5 / #6** (rigid frame, one in / one out). Different from the count/budget *variable* windows #3 / #4 / #7 / #8 — don't cross the templates.
