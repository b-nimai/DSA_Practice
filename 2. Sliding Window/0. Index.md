## Variants of Sliding Window (cluster the problems)

| Variant | Problems | Signature |
|---------|----------|-----------|
| **Fixed-size window** | 1, 5, 6, 9 | width given; slide one element in / one out per step |
| **Variable-size, shrink-while-valid** (shortest) | 2 | grow `right`; `while (valid) { record; left++ }` |
| **Variable-size, shrink-while-invalid** (longest) | 3, 4, 7, 8 | grow `right`; `while (invalid) { left++ }`; record `right-left+1` |
| **Variable-size, slide-by-one** (longest, window grows ‚Č§1/step) | 4, 7, 8 | grow `right`; `if (invalid) left++`; window only slides forward ‚ÄĒ smaller windows can never beat current max; answer = `n - left` |
| **Window + counts/freq map** | 3, 4, 5, 6, 7, 8 | `unordered_map`/array of counts; condition on distinct / replacement / flip budget / anagram match |

## When to reach for Sliding Window
1. Asking for **longest / shortest / max / min** of a **contiguous** subarray or substring under a constraint.
2. Brute force is "try every window" ‚ÄĒ and a window's validity changes monotonically as you grow/shrink it.
3. Keywords: "at most K", "exactly K", "no repeating", "‚Č§ k replacements", "fixed length k".
4. All elements positive (so growing only increases the running sum) ‚ÄĒ common precondition for the sum-based variants.

## Common Pitfalls Across the Pattern
- **Fixed vs variable**: "exactly k" ‚Üí fixed window; "shortest/longest such that‚Ä¶" ‚Üí variable. Don't mix the templates.
- Shrink with **`while`**, not `if` ‚ÄĒ keep contracting as long as the condition warrants.
- "Shortest" ‚Üí record length *while still valid*, then shrink. "Longest" ‚Üí shrink *until valid again*, then record.
- Impossible-answer sentinel: use `n + 1` for min-length (map back to 0); `0` for max-length.
- `nums.size()` is unsigned ‚ÄĒ cast to `int` before subtracting (`i - k`).
- The sum-based shrink trick **requires non-negative elements**; with negatives the window approach breaks.
- Watch overflow on the running sum if value/length bounds are large ‚Üí `long long`.

## C++ Sliding Window templates
```cpp
// Fixed size k
long sum = 0;
for (int i = 0; i < k; i++) sum += a[i];
long best = sum;
for (int i = k; i < n; i++) { sum += a[i] - a[i - k]; best = max(best, sum); }

// Variable size ‚ÄĒ shortest valid window
int left = 0, best = n + 1; long sum = 0;
for (int right = 0; right < n; right++) {
    sum += a[right];
    while (/* window valid */) { best = min(best, right - left + 1); sum -= a[left++]; }
}

// Variable size ‚ÄĒ longest valid window
int left = 0, best = 0;
for (int right = 0; right < n; right++) {
    /* add a[right] to window state */
    while (/* window invalid */) { /* remove a[left] */ left++; }
    best = max(best, right - left + 1);
}
```
