# 8. 230. Kth Smallest Element in a BST (LC 230) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/kth-smallest-element-in-a-bst/)

**Pattern:** Tree DFS — **in-order traversal of a BST visits values in sorted order**; the kth node you touch is the answer
**Trigger:** "**kth smallest** (or kth largest, reverse in-order) value in a BST" → in-order DFS with a **shared countdown** `k`; stop and return the moment `--k == 0`

## Problem (1-liner)
Return the **kth smallest** (1-indexed) value among all nodes of a BST.
**Examples:** `[3,1,4,null,2], k=1` → `1` · `[5,3,6,2,4,null,null,1], k=3` → `3`
**Constraints:** `1 ≤ k ≤ n ≤ 10^4`, `0 ≤ node.val ≤ 10^4`.
**Follow-up:** if the tree is modified often and kth queries are frequent → augment each node with its **subtree size** (or keep an order-statistics structure) so a query is O(h) instead of O(k).

## Idea (1-liner)
A BST's **in-order** walk (left → node → right) emits values in ascending order. So the answer is just the kth value produced by that walk. Carry `k` as a **shared counter by reference**, decrement it as you "visit" each node, and the node where `k` hits `0` is the kth smallest — return it straight up the call stack and short-circuit the rest.

## Approach
1. **Recurse left first** — smaller values come before the current node in sorted order.
2. **If the left side already found the answer, bubble it up** unchanged (don't keep counting).
3. **Visit the node:** `--k`; if it reached `0`, this node is the kth smallest → return `root->val`.
4. **Otherwise recurse right** and bubble up its result.
5. **Sentinel `-1` for "not found here"** — propagated up so a parent knows whether a subtree already produced the answer.
6. **`k` must be shared across the whole traversal** — pass it by reference (`int &k`) so decrements from the left subtree persist into the node and the right subtree.

> **The bug that bit you:** the helper recursed into the *public* `kthSmallest`, whose `k` is **by value**. Each call got a fresh copy, so the left subtree's decrements never reached the node or the right subtree — the global count was lost. Fix: recurse into the **helper** so every node shares the same `int &k`.

## Code (clean canonical)
```cpp
class Solution {
    int helper(TreeNode* root, int& k) {
        if (!root) return -1;                       // not found in this subtree

        int left = helper(root->left, k);           // in-order: left first
        if (left != -1) return left;                // answer already found below

        if (--k == 0) return root->val;             // this is the kth visited node

        return helper(root->right, k);              // else look right (returns -1 if absent)
    }
public:
    int kthSmallest(TreeNode* root, int k) {
        return helper(root, k);                      // share one k by reference
    }
};
```
**TC:** O(h + k) — descend to the smallest (`h`) then visit `k` nodes; worst case O(n).  **SC:** O(h) recursion stack.

Same O(h+k)/O(h).

> ⚠️ **`-1` as the sentinel is only safe because `0 ≤ node.val`.** If node values could be negative, `-1` would collide with a real value — use a `bool found` flag + out-param, or a class member `result`, instead.

## Other Approaches
- **Iterative in-order with an explicit stack** — push left spine, pop a node (that's the next-smallest), decrement `k`; when `k == 0` return it. Avoids recursion depth on a skewed tree and naturally stops early. O(h+k)/O(h).
- **Collect-then-index** — in-order push everything into a vector, return `v[k-1]`. Simple but O(n) time *and* space and no early stop; fine for a one-off, wasteful for repeated queries.
- **Follow-up (frequent insert/delete):** augment nodes with **subtree size**. Then kth-smallest is O(h): if `leftSize + 1 == k` it's this node; if `k ≤ leftSize` go left; else go right with `k -= leftSize + 1`. An AVL/Red-Black order-statistics tree keeps it O(log n).

## Pitfalls
- **Share `k` by reference** — the bug you hit. Recursing into a by-value `k` (or resetting it) loses the global count; every node in the traversal must decrement the *same* counter (`int &k`, or a member variable).
- **Recurse into the right helper** — calling the public entry point instead of the helper silently re-takes `k` by value. Make sure both recursive calls hit the `int &k` overload.
- **Sentinel must not collide with a real value** — `-1` works only because values are `≥ 0`. With possible negatives, use a `found` flag instead.
- **Stop early — don't keep walking after found** — once a subtree returns a real value, return it immediately; continuing to recurse wastes time and (with a shared `k`) can corrupt nothing here but defeats the O(h+k) bound.
- **In-order direction defines smallest vs largest** — left→node→right gives ascending (kth *smallest*). Don't accidentally pre-order; the sorted property only holds for **in-order**.
