# 3. 226. Invert Binary Tree (LC 226) — Easy
🔗 [Open on LeetCode](https://leetcode.com/problems/invert-binary-tree/)

**Pattern:** Tree DFS — **mutate-in-place traversal** (swap children at every node, recurse into both)
**Trigger:** "mirror / flip / invert the tree left↔right" → at each node swap its two children, then recurse into both subtrees; order (pre vs post) doesn't matter because the swap and the two recursions touch independent subtrees

## Problem (1-liner)
Given the `root` of a binary tree, **mirror it** — every node's left and right subtree swap — and return the (same) root.
**Examples:** `[4,2,7,1,3,6,9]` → `[4,7,2,9,6,3,1]` · `[2,1,3]` → `[2,3,1]` · `[]` → `[]`
**Constraints:** `0 … 100` nodes, `-100 ≤ node.val ≤ 100`.

## Idea (1-liner)
A mirror image is defined recursively: invert(node) = swap its two children, then invert each. One swap per node, visited once ⇒ **O(n)**. Same DFS skeleton as [[1-104-maximum-depth-of-binary-tree|max depth]] / [[2-100-same-tree|same tree]], but instead of *returning info up* you **rewire pointers on the way down** — the return value is just the unchanged root.

## Approach
1. **Null guard** ⇒ return — an empty subtree is its own mirror; this is the base case at every leaf's child.
2. **Swap the two child pointers** — `swap(node->left, node->right)`. This single line *is* the work; everything else is plumbing.
3. **Recurse into both children** — `invert(node->left); invert(node->right);`. After the swap, `left` already holds the old right subtree, but that's fine: each subtree gets inverted exactly once regardless.
4. **Return `root`** — the tree is mutated in place; you hand back the same pointer.

## Code (clean canonical)
```cpp
class Solution {
public:
    TreeNode* invertTree(TreeNode* root) {
        if (!root) return nullptr;            // empty subtree is its own mirror
        std::swap(root->left, root->right);   // the entire "work": flip children
        invertTree(root->left);               // recurse both (order irrelevant)
        invertTree(root->right);
        return root;
    }
};
```
**TC:** O(n) — each node visited once, O(1) work per node.  **SC:** O(h) recursion stack — O(log n) balanced, O(n) skewed.

## My Solution
Correct and clean — split into a `void invertTreeHlper` plus a public wrapper. That works, but the wrapper is **redundant**: `invertTree` already has the recursive signature `TreeNode* → TreeNode*`, so it can recurse on itself with no helper. Two nits to tighten it: (1) replace the manual 3-line temp swap (`leftNode = …; left = right; right = leftNode;`) with **`std::swap(root->left, root->right)`** — same cost, one line, no naming; (2) typo `invertTreeHlper` → `invertTreeHelper` (moot once the helper is gone). Algorithm and complexity are already optimal.

## Other Approaches
- **Swap the *results* of recursion** (functional style): `auto l = invertTree(root->left); auto r = invertTree(root->right); root->left = r; root->right = l;`. Reads nicely, same O(n)/O(h). **Trap:** you must capture *both* results into locals first — writing `root->left = invertTree(root->right)` directly clobbers `root->left` before the second call reads it, inverting the wrong subtree.
- **Iterative BFS/DFS** — push `root` onto a queue/stack; pop a node, `swap` its children, push the (non-null) children. O(n) time, O(n) space for the queue. Use when recursion depth is a worry (not here, n ≤ 100). The swap logic is identical; only the traversal driver changes.

## Pitfalls
- **Don't forget to recurse after swapping** — swapping only the root's two children flips just the top level; the subtrees stay un-mirrored. The recursion is what makes it a full inversion.
- **Functional form needs both results saved first** — `root->left = invertTree(root->right)` then reading `root->left` again reads the *already-overwritten* pointer. Capture into `l`/`r` before assigning (see above).
- **Return the root, not void into the wild** — the signature must hand back `root`; the tree is mutated in place, so returning the same pointer is all that's needed (return `nullptr` on the empty base case).
- **Order is a non-issue** — pre-order, post-order, or swap-between-recursions all produce the same mirror, because the swap at a node and the inversions of its two subtrees are independent. Unlike [[2-100-same-tree|Same Tree]], there's no base-case *ordering* trap here.
