# Tree DFS — Index

60-second whole-pattern recall. Read the **Trigger** column and try to recall the **Idea** before peeking.

**Core skeleton:** `if (!node) return base; L = dfs(left); R = dfs(right); return combine(L, R, node);` — work happens **post-order** (after children return), and the return value is the info the parent needs.

## When to reach for Tree DFS
1. You need a **path / sum / depth / diameter** in a binary tree, or to **validate** a structural property.
2. The answer at a node depends on **information combined from both subtrees** — recurse, then merge `L` and `R`.
3. Reach for **BFS** instead when you need per-level info or **minimum** depth (BFS short-circuits at the first leaf).

## Common Pitfalls Across the Pattern
- **Base case value** — an empty child usually returns `0` (depth) or a neutral sentinel; returning the wrong base shifts every count by one.
- **`max(L, R)` vs `L + R`** — depth/height takes the deeper subtree (`max`); diameter-style "path through this node" sums both (`L + R`). Know which the question wants.
- **Nodes vs edges** — depth counted in nodes makes a single node `1`; in edges it's `0`. Re-read the definition each problem.
