## Variants of Binary Search (cluster the problems)

| Variant | Problems | Signature |
|---------|----------|-----------|
| **Exact match** (closed interval, return index or âˆ’1) | 704 | `[low, high]`, `while (low <= high)`, `mid Â± 1` on both sides so the window strictly shrinks; on hit `return mid`, after the loop `return -1`. Use when the target must *exist* and you want its precise index |
| **Lower bound / first-true** (return the boundary / insertion point) | 35, 278 | smallest index `i` where the monotonic predicate flips true; half-open window + `while (low < high)` + `high = mid` (no `-1`), return `low`. Predicate can be a value compare (35) **or an API/feasibility check** (278). The reusable seed for binary-search-on-answer |
| **Rotated array** (not globally sorted, exact index) | 33 | closed interval `while (low <= high)`; each step `nums[low] <= nums[mid]` tells you the sorted half, then a range-check decides which side to keep. At least one half is always sorted â€” that invariant is what restores O(log n) on a rotated array |
| **Rotated array â€” find the pivot** (the minimum, no target) | 153 | half-open `while (low < high)`; compare `nums[mid]` to `nums[high]` and converge: `>` â‡’ `low = mid+1`, else `high = mid` (no `-1`, `mid` is a live candidate); return `nums[low]`. The min is the rotation point â€” walk toward it instead of range-checking a target |
| **Boundary search** (first/last index in a block of duplicates) | 34 | the exact-match loop run twice, but **don't return on a hit** â€” record `mid` and keep shrinking: `high = mid-1` for the first occurrence, `low = mid+1` for the last. The `==` tie-break direction is the only thing that changes between the two passes |
| **Slope search** (binary search on an *unsorted* array) | 162 | no sorted order to exploit â€” compare `nums[mid]` to a **neighbor** and walk uphill: half-open `while (low < high)`, `nums[mid] > nums[mid+1] ? high = mid : low = mid+1`, return `low`. The `-âˆž` boundary makes a peak unavoidable, so "walk toward the larger neighbor" always converges. Same `high = mid` / `low = mid+1` mechanics as 153, but the signal is the local slope, not a global pivot |

## When to reach for Binary Search
1. Input is **sorted** (or rotated-sorted) and you need an index / boundary in O(log n).
2. A **monotonic predicate** "smallest/largest X such that `check(X)` is true" â€” even with no array to search, you binary-search the *answer range* (Koko 875, Split Array 410).
3. "**Minimize the maximum**" / "**maximize the minimum**" phrasing â€” the classic binary-search-on-answer tell.
4. "First/last position", "insert position", "find peak" â€” all lower-bound loops with the comparison swapped.

## Common Pitfalls Across the Pattern
- **`<` vs `<=` must match the interval.** Closed `[low, high]` â‡’ `while (low <= high)` + `high = mid - 1`. Half-open `[low, high)` â‡’ `while (low < high)` + `high = mid`. Mixing the pieces either skips the last element or infinite-loops.
- **Always shrink past `mid`.** In the exact-match loop use `mid Â± 1` on the discarded side; forgetting the `Â±1` (e.g. `high = mid` in the closed loop) infinite-loops on a size-1/2 window.
- **Overflow-safe midpoint.** `mid = low + (high - low) / 2`, never `(low + high) / 2`. Free correct habit; mandatory once you binary-search a value range instead of indices.
- **The post-loop return encodes the answer.** Exact-match returns `-1`; lower-bound returns `low` (count of elements `< target` = insertion point). Don't copy `return -1` into an insert-position problem.
- **High bound for lower-bound is `n`, not `n-1`.** The insertion point can legitimately be past the end.

## C++ Binary Search templates
```cpp
// Exact match â€” Binary Search (LC 704)
int search(vector<int>& nums, int target) {
    int low = 0, high = nums.size() - 1;           // closed [low, high]
    while (low <= high) {
        int mid = low + (high - low) / 2;
        if (nums[mid] == target) return mid;
        else if (nums[mid] > target) high = mid - 1;
        else low = mid + 1;
    }
    return -1;
}

// Lower bound / insertion point â€” Search Insert Position (LC 35)
int searchInsert(vector<int>& nums, int target) {
    int low = 0, high = nums.size();               // half-open [low, n)
    while (low < high) {
        int mid = low + (high - low) / 2;
        if (nums[mid] < target) low = mid + 1;     // too small, exclude
        else high = mid;                           // >= target, keep as candidate
    }
    return low;                                    // first index with nums[i] >= target
}

// Binary search on the answer â€” generic monotonic predicate
int minFeasible(int lo, int hi) {                  // smallest x in [lo, hi] with check(x) true
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (check(mid)) hi = mid;
        else lo = mid + 1;
    }
    return lo;
}
```
