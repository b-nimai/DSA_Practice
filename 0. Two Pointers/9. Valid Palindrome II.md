# 9. Valid Palindrome II (LC 680) — Easy
🔗 [Open on LeetCode](https://leetcode.com/problems/valid-palindrome-ii/)

**Pattern:** Two Pointers — opposite ends + branch on first mismatch (≤ 1 deletion)
**Trigger:** palindrome / symmetry with at most one fix-up allowed

## Problem (1-liner)
Return `true` if deleting **at most one** character can make `s` a palindrome.
**Example:** `"abca"` → `true (delete 'c')`; `"abc"` → `false`

## Idea (1-liner)
Walk inward; on first mismatch, you have exactly two recovery options — skip left or skip right — and the substring after that skip must be a plain palindrome.

## Approach
1. `l = 0`, `r = n - 1`.
2. While `l < r`:
   - `s[l] == s[r]` → `l++; r--`
   - else → return `isPal(l+1, r) || isPal(l, r-1)`
3. Return `true`.

## Code (clean canonical)
```cpp
class Solution {
    bool isPal(const string& s, int l, int r) {
        while (l < r) {
            if (s[l] != s[r]) return false;
            l++; r--;
        }
        return true;
    }
public:
    bool validPalindrome(string s) {
        int l = 0, r = s.size() - 1;
        while (l < r) {
            if (s[l] != s[r])
                return isPal(s, l + 1, r) || isPal(s, l, r - 1);
            l++; r--;
        }
        return true;
    }
};
```
**TC:** O(n)  **SC:** O(1)

## Other Approaches
- **Try every single deletion:** for each `i`, check if `s` without index `i` is a palindrome → O(n²) / O(n)
- **DP (edit distance ≤ 1 vs reverse):** overkill, O(n²) / O(n²)

## Pitfalls
- On mismatch, **both** branches must be tried (`l+1..r` OR `l..r-1`); short-circuit with `||`
- Pass string by const-ref in helper to avoid copy cost
- Empty / single-char strings are always palindromes
- Generalization (LC 1216, ≤ k deletions) needs a different approach — DP
