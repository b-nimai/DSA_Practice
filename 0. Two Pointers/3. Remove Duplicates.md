# 3. Remove Duplicates from Sorted Array (LC 26) — Easy
🔗 [Open on LeetCode](https://leetcode.com/problems/remove-duplicates-from-sorted-array/)

**Pattern:** Two Pointers — slow/fast (write index + scan index)
**Trigger:** in-place dedup on sorted array, return new length

## Problem (1-liner)
Sorted array → remove dups in-place; first `k` elements unique; return `k`.
**Example:** `[1,1,2]` → `k=2`, `nums=[1,2,_]`

## Idea (1-liner)
`write` lags `read`. Copy `nums[read]` to `nums[write]` only when it differs from previous written.

## Approach
1. If `n <= 1` → return `n`.
2. `write = 1` (first element always kept).
3. For `read = 1..n-1`:
   - If `nums[read] != nums[write - 1]` → `nums[write++] = nums[read]`.
4. Return `write`.

## Code (clean canonical)
```cpp
int removeDuplicates(vector<int>& nums) {
    if (nums.empty()) return 0;
    int write = 1;
    for (int read = 1; read < nums.size(); read++) {
        if (nums[read] != nums[write - 1])
            nums[write++] = nums[read];
    }
    return write;
}
```
**TC:** O(n)  **SC:** O(1)

## My Solution
```cpp
class Solution {
public:
    int removeDuplicates(vector<int>& nums) {
        if(nums.size() <= 1) return nums.size();
        int start = 1;
        for(int i = 1; i < nums.size(); i++) {
            if(nums[start - 1] != nums[i]) {
                nums[start] = nums[i];
                start++;
            }
        }
        return start;
    }
};
```

## Other Approaches
- **Set / extra array:** copy uniques to new container, write back → O(n) / O(n)
- **`std::unique` + `erase`:** library shortcut → O(n) / O(1) (not interview-friendly)

## Pitfalls
- Compare `nums[read]` to `nums[write - 1]` (last written), not `nums[read - 1]`
- Empty array edge case
- Variant LC 80 (allow ≤ 2 dups) — check `nums[write - 2]` instead
