# 1. Two Sum II (LC 167) — Easy
🔗 [Open on LeetCode](https://leetcode.com/problems/two-sum-ii-input-array-is-sorted/)

**Pattern:** Two Pointers — opposite ends on sorted array
**Trigger:** sorted array + find pair with target sum

## Problem (1-liner)
1-indexed sorted array; return `[i+1, j+1]` such that `nums[i] + nums[j] == target`.
**Example:** `[2,7,11,15], t=9` → `[1,2]`

## Idea (1-liner)
`l, r` at ends. Sum too small → `l++`; too big → `r--`; equal → done.

## Approach
1. `l = 0`, `r = n - 1`
2. Loop while `l < r`:
   - `sum == target` → return `{l+1, r+1}`
   - `sum < target` → `l++`
   - else → `r--`

## Code (clean canonical)
```cpp
vector<int> twoSum(vector<int>& nums, int target) {
    int l = 0, r = nums.size() - 1;
    while (l < r) {
        int s = nums[l] + nums[r];
        if (s == target) return {l + 1, r + 1};
        (s < target) ? l++ : r--;
    }
    return {};
}
```
**TC:** O(n)  **SC:** O(1)

## My Solution
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& numbers, int target) {
        int start = 0, last = numbers.size() - 1;
        while(start < last) {
          int sum = numbers[start] + numbers[last];
          if(sum == target) return {start + 1, last + 1};
          else if(sum < target) start++;
          else last--;
        }
        return {};
    }
};
```

## Other Approaches
- **Brute force:** nested loops, check every pair → O(n²) / O(1)
- **HashMap:** store `target - nums[i]` → O(n) / O(n) (ignores sorted property)
- **Binary search:** for each `i`, binary search complement → O(n log n) / O(1)

## Pitfalls
- Output is **1-indexed**, not 0-indexed
- Cannot reuse the same element (`l < r`, not `l <= r`)
- Watch overflow if values are large → use `long long` for sum
