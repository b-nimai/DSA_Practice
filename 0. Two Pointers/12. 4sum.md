# 12. 4Sum (LC 18) ‚Äî Medium
üîó [Open on LeetCode](https://leetcode.com/problems/4sum/)

**Pattern:** Two Pointers ‚Äî two fixed outer loops + inner two-pointer, skip duplicates at every level
**Trigger:** unique k-tuples summing to a target; reduce k-sum to (k-1)-sum

## Problem (1-liner)
Return all **unique** quadruplets `[a, b, c, d]` (distinct indices) with `a + b + c + d == target`.
**Example:** `nums = [1,0,-1,0,-2,2]`, `target = 0` ‚Üí `[[-2,-1,1,2], [-2,0,0,2], [-1,0,0,1]]`

## Idea (1-liner)
Sort. Fix `i`, fix `j`, then run the classic two-pointer on the remaining suffix for `target - nums[i] - nums[j]`. Skip duplicates after `i`, after `j`, and after each match on `l`/`r`. Use 64-bit sums ‚Äî four `int`s can overflow.

## Approach
1. Sort `nums`. `n = nums.size()`.
2. For `i = 0..n-4`: skip if `i > 0 && nums[i] == nums[i-1]`.
3. For `j = i+1..n-3`: skip if `j > i+1 && nums[j] == nums[j-1]`.
4. `l = j+1`, `r = n-1`.
   - While `l < r`, `sum = nums[i] + nums[j] + nums[l] + nums[r]` (as `long long`):
     - `sum < target` ‚Üí `l++`
     - `sum > target` ‚Üí `r--`
     - `sum == target` ‚Üí push `{nums[i], nums[j], nums[l], nums[r]}`, then `l++; r--;` and
       `while (l < r && nums[l] == nums[l-1]) l++;` / `while (l < r && nums[r] == nums[r+1]) r--;`

## Code (clean canonical)
```cpp
vector<vector<int>> fourSum(vector<int>& nums, int target) {
    sort(nums.begin(), nums.end());
    int n = nums.size();
    vector<vector<int>> res;
    for (int i = 0; i < n - 3; i++) {
        if (i > 0 && nums[i] == nums[i - 1]) continue;
        for (int j = i + 1; j < n - 2; j++) {
            if (j > i + 1 && nums[j] == nums[j - 1]) continue;
            int l = j + 1, r = n - 1;
            while (l < r) {
                long long sum = 1LL * nums[i] + nums[j] + nums[l] + nums[r];
                if (sum < target) l++;
                else if (sum > target) r--;
                else {
                    res.push_back({nums[i], nums[j], nums[l], nums[r]});
                    l++; r--;
                    while (l < r && nums[l] == nums[l - 1]) l++;
                    while (l < r && nums[r] == nums[r + 1]) r--;
                }
            }
        }
    }
    return res;
}
```
**TC:** O(n¬≥)  **SC:** O(1) extra (output not counted)

## My Solution
```cpp
class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        sort(nums.begin(), nums.end());
        vector<vector<int>> result;
        int size = nums.size();

        for(int i = 0; i < size - 3; i++) {
          if(i > 0 && nums[i] == nums[i - 1]) continue;

          for(int j = i + 1; j < size - 2; j++) {
            if(j > i + 1 && nums[j] == nums[j - 1]) continue;

            int start = j + 1, end = size - 1;
            while(start < end) {
              long long sum = 1LL * nums[i] + nums[j] + nums[start] + nums[end];
              if(sum > target) end--;
              else if(sum < target) start++;
              else {
                result.push_back({nums[i], nums[j], nums[start], nums[end]});
                start++;
                end--;
                while(start < end && nums[start] == nums[start - 1]) start++;
                while(start < end && nums[end] == nums[end + 1]) end--;
              }
            }
          }
        }
        return result;
    }
};
```

> Correct and clean. Optional speedups: prune with `1LL*nums[i]+nums[i+1]+nums[i+2]+nums[i+3] > target` ‚Üí break; `1LL*nums[i]+nums[n-3]+nums[n-2]+nums[n-1] < target` ‚Üí continue (same idea on the `j` loop).

## Other Approaches
- **Brute force:** 4 nested loops + set-dedup ‚Üí O(n‚Å¥) / O(n)
- **Hashset on the last pair:** two outer loops + hashset for the inner two ‚Üí O(n¬≥) / O(n)
- **Generic kSum recursion:** recurse k ‚Üí 2, then two-pointer base case ‚Üí O(n^(k-1))

## Pitfalls
- **Overflow:** `nums[i]+nums[j]+nums[l]+nums[r]` can exceed `int`. Cast the first operand (`1LL * ...`) *before* adding, and compare against `target` (or make `target` a `long long`).
- Dedup at **3 places** beyond `i`: after `i`, after `j` (`j > i+1`, not `j > 0`), and on both `l`/`r` after a match.
- Loop bounds: `i < n-3`, `j < n-2`, so there's always room for `l < r`.
- `nums.size()` is unsigned ‚Äî store it in an `int` (`int n = nums.size();`) before doing `n - 3`.
