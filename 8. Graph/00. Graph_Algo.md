# Topological Sort (DAG ordering)

**Pattern:** Order the vertices of a **DAG** so every edge `u → v` has `u` before `v`.
**Trigger:** "dependencies / prerequisites / build order / course schedule / task ordering" on a **directed acyclic** graph → topo sort (DFS post-order **or** Kahn's BFS).

## Idea (1-liner)
A valid linear order exists **iff the graph is a DAG**. Two ways to get it: **DFS** — push a node *after* all its descendants, then reverse (post-order). **Kahn's BFS** — repeatedly emit nodes with **in-degree 0**, peeling them off the front.

> **Load-bearing insight:** DFS pushes **on the way out** (post-order) → the stack, reversed, is the topo order. Kahn peels **in-degree-0** nodes → the queue order *is* the topo order. Same result, opposite mechanics.

---

## Approach A — DFS (post-order + reverse)
1. `visited[]` all 0. For every unvisited node, run DFS.
2. In DFS: mark visited, recurse into all unvisited neighbours, then **push the node onto a stack** (after the loop — post-order).
3. Pop the whole stack → that reversed post-order is a valid topological order.

### Code (clean canonical)
```cpp
class Solution {
    void dfs(int node, vector<int>& visited, vector<int> adj[], stack<int>& st) {
        visited[node] = 1;
        for (int nbr : adj[node])
            if (!visited[nbr]) dfs(nbr, visited, adj, st);
        st.push(node);                 // push AFTER children → post-order
    }
public:
    vector<int> topoSort(int V, vector<int> adj[]) {
        vector<int> visited(V, 0);
        stack<int> st;
        for (int i = 0; i < V; i++)
            if (!visited[i]) dfs(i, visited, adj, st);   // cover all components

        vector<int> topo;
        while (!st.empty()) { topo.push_back(st.top()); st.pop(); }
        return topo;
    }
};
```
**TC:** O(V + E) — each node/edge once.  **SC:** O(V) — visited + stack + recursion depth (up to O(V)).

---

## Approach B — Kahn's Algorithm (BFS, in-degree)
1. Compute `indegree[]` for every node (count incoming edges).
2. Push all **in-degree-0** nodes into a queue (no prerequisites → can go first).
3. Pop a node → append to `topo`; for each neighbour, `--indegree`; if it hits 0, push it.
4. Repeat until the queue empties.

### Code (clean canonical)
```cpp
class Solution {
public:
    vector<int> topoSort(int V, vector<int> adj[]) {
        vector<int> indegree(V, 0);
        for (int i = 0; i < V; i++)
            for (int node : adj[i]) indegree[node]++;

        queue<int> q;
        for (int i = 0; i < V; i++)
            if (indegree[i] == 0) q.push(i);

        vector<int> topo;
        while (!q.empty()) {
            int node = q.front(); q.pop();
            topo.push_back(node);
            for (int nbr : adj[node])
                if (--indegree[nbr] == 0) q.push(nbr);
        }
        return topo;                  // if topo.size() < V → graph has a CYCLE
    }
};
```
**TC:** O(V + E).  **SC:** O(V) — indegree + queue.

## DFS vs Kahn — which to reach for
| | DFS post-order | Kahn's BFS |
|---|---|---|
| Mechanic | push on exit, reverse | peel in-degree-0 |
| Cycle detection | needs a 3rd "in-recursion" colour | **free** — `topo.size() < V` ⇒ cycle |
| Order flavour | reverse finish-time | natural "levels" / lexicographic if you use a min-heap |
| Risk | recursion depth (deep graphs overflow) | iterative, no stack risk |
| Default pick | quick to write when a DAG is guaranteed | **preferred when you must detect a cycle** (Course Schedule) |

## Pitfalls
- **Only defined on a DAG.** A cycle ⇒ no valid order. Plain DFS above **won't detect it** — you'd need a `pathVisited`/recursion-stack state. Kahn detects it for free (`topo.size() < V`).
- **DFS must push AFTER the neighbour loop** (post-order). Push before → wrong order.
- **Answer isn't unique** — many valid topo orders; judges accept any valid one.
- **Cover every component** — loop `i = 0..V-1` and start from each unvisited / in-degree-0 node, not just node 0 (graph may be disconnected).
- **VLA trap:** `int visited[V] = {0}` where `V` is runtime → non-standard variable-length array and the `{0}` init isn't guaranteed. Use `vector<int> visited(V, 0)`.
- **Kahn: decrement then test** — `if (--indegree[nbr] == 0)`; comparing before decrement misses ready nodes.
