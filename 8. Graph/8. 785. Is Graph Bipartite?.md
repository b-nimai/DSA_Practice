# 8. 785. Is Graph Bipartite? (LC 785) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/is-graph-bipartite/)

**Pattern:** Graph — **2-coloring** (bipartite check via DFS/BFS)
**Trigger:** "can the nodes be split into **two groups** so **every edge crosses** between them (no edge inside a group)?" → try to **2-color** the graph; a **same-colored edge** (an **odd cycle**) means **not bipartite**

## Problem (1-liner)
Undirected graph as adjacency list `graph[u]` (may be **disconnected**, no self/parallel edges). Return `true` iff nodes split into two sets `A`, `B` with **every edge between** `A` and `B`.
**Examples:** `[[1,2,3],[0,2],[0,1,3],[0,2]]` → `false` (0–1–2 forms an odd cycle) · `[[1,3],[0,2],[1,3],[0,2]]` → `true` (`{0,2}` / `{1,3}`).
**Constraints:** `1 ≤ n ≤ 100`, `0 ≤ graph[u].length < n`, values unique, undirected. Single node / isolated nodes ⇒ trivially bipartite.

## Idea (1-liner)
**Bipartite ⇔ 2-colorable ⇔ no odd cycle.** Color a start node `0`, force every neighbour to the **opposite** color; if you ever meet a neighbour **already colored the same**, an odd cycle exists → `false`. Wrap in an outer loop over all nodes so **disconnected components** are each seeded.

> **Load-bearing insight:** the only failure is a neighbour already carrying **your own** color. Flip color on every edge (`!color`); an odd-length cycle makes that flip return to the start with the wrong parity — that's the contradiction. Even cycles color fine.

## Approach (DFS — default)
1. `colors[n]` all `-1` (uncolored). Loop `i = 0…n-1`.
2. For each still-uncolored `i`, run `possible(i, color = 0)` (new component).
3. In `possible`: paint `colors[node] = color`; for each neighbour —
   - **uncolored** → recurse with `!color`; propagate `false` up on conflict.
   - **already colored == node's color** → same-color edge ⇒ return `false`.
4. All components color cleanly ⇒ `true`.

## Code (clean canonical — DFS)
```cpp
class Solution {
    bool possible(vector<vector<int>>& graph, int node, vector<int>& colors, int color) {
        colors[node] = color;
        for (int nbr : graph[node]) {
            if (colors[nbr] == -1) {                    // uncolored → paint opposite
                if (!possible(graph, nbr, colors, !color)) return false;
            } else if (colors[nbr] == color) return false;   // same color = odd cycle
        }
        return true;
    }
public:
    bool isBipartite(vector<vector<int>>& graph) {
        int n = graph.size();
        vector<int> colors(n, -1);
        for (int i = 0; i < n; ++i)                     // seed every component
            if (colors[i] == -1 && !possible(graph, i, colors, 0)) return false;
        return true;
    }
};
```
**TC:** O(V + E) — each node colored once, each edge scanned twice.  **SC:** O(V) — `colors` + recursion stack.

## My Solution (both correct ✅)
Both traversals implemented, both right.

**DFS** *(canonical block above)* — the branch logic is subtle but sound: the `else if (colors[nbr] == color)` only fires when `nbr` is already colored (the `== -1` guard failed first), so there's no double-evaluation. `!color` flips `0↔1` cleanly.

**BFS** — seed the start, and on discovery of an uncolored neighbour assign `!colors[curr]`; reject on a same-color neighbour:
```cpp
bool possible(vector<vector<int>>& graph, int node, vector<int>& colors, int color) {
    colors[node] = color;
    queue<int> q; q.push(node);
    while (!q.empty()) {
        int curr = q.front(); q.pop();
        for (int nbr : graph[curr]) {
            if (colors[nbr] == -1) { colors[nbr] = !colors[curr]; q.push(nbr); }
            else if (colors[nbr] == colors[curr]) return false;   // same color = odd cycle
        }
    }
    return true;
}
```
*Nit:* the BFS `possible` takes a `color` param but seeds via `colors[node] = color` — harmless; you could drop it and seed `0` directly. Both O(V+E) / O(V).

## Other Approaches
- **DFS 2-coloring** *(default)* — recursive, tiny code, natural conflict propagation. Watch recursion depth on huge graphs (here `n ≤ 100`, safe). O(V+E) / O(V).
- **BFS 2-coloring** *(your alt, equally optimal)* — iterative, no stack-depth risk; reach for it when the graph could be deep. O(V+E) / O(V).
- **Union-Find** — for each node, union all its neighbours together (they must share a side); if a node ends up in the **same set as one of its own neighbours**, not bipartite. Slightly more code, useful when edges arrive dynamically. ~O(V+E·α) / O(V).
- **Brute force (no contender)** — try every 2-way partition and check edges: O(2ⁿ). Mention only to motivate coloring.

## Pitfalls
- **Must loop over all nodes** — the graph is **disconnected**; a single DFS/BFS misses whole components. (Same outer-loop discipline as [[6. 207. Course Schedule]] / [[7. 210. Course Schedule II]].)
- **Conflict test is same-color, not just "visited"** — a colored neighbour is fine if it's the **opposite** color; only `colors[nbr] == curr's color` is the failure.
- **`-1` sentinel ≠ a color** — colors are `0/1`; keep uncolored distinct (`-1`) so `!color` never collides with "unvisited".
- **DFS: propagate the `false`** — a deep conflict must bubble all the way up; returning `true` early hides it.
- **Bipartite = no odd cycle** — even cycles are fine; don't reject on *any* cycle.
