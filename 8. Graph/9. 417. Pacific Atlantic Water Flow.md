# 9. 417. Pacific Atlantic Water Flow (LC 417) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/pacific-atlantic-water-flow/)

**Pattern:** Graph flood-fill — **multi-source, border-anchored DFS/BFS** (flood *inward from each ocean's edge*, then intersect the two reachable sets)
**Trigger:** "cells from which water can reach **both** oceans / two targets on opposite borders" → don't simulate flow from every cell outward — **flood inward from each border, mark what each ocean reaches, then AND the two sets**

## Problem (1-liner)
`m × n` grid of heights; Pacific touches **top + left** edges, Atlantic touches **bottom + right**. Water flows from a cell to a 4-neighbour of **height ≤** it. Return every cell `[r,c]` from which water can reach **both** oceans.
**Examples:** `[[1,2,2,3,5],[3,2,3,4,4],[2,4,5,3,1],[6,7,1,4,5],[5,1,1,2,4]]` → `[[0,4],[1,3],[1,4],[2,2],[3,0],[3,1],[4,0]]` · `[[1]]` → `[[0,0]]` (the lone cell touches both).
**Constraints:** `1 ≤ m, n ≤ 200`, `0 ≤ heights[r][c] ≤ 10^5`.

## Idea (1-liner)
Asking "can *this* cell reach an ocean?" per-cell repeats work. **Reverse the flow:** start *at* each ocean's border and climb to cells of height **≥** the current one — those are exactly the cells that could drain *into* that ocean. Do it once for Pacific, once for Atlantic; the answer is the **intersection**.

> **Load-bearing insight:** flood **inward from the border**, and **invert the height test** — going ocean→land you may only step to a cell that is **as high or higher**, because the real (land→ocean) flow needs height to be non-increasing. Same border-anchored trick as [[4-130-surrounded-regions|Surrounded Regions]].

## Approach
1. Two `visited` grids, `pac` and `atl` (a cell can belong to both).
2. **Seed Pacific** from the top row + left column; **seed Atlantic** from the bottom row + right column.
3. From each seed run DFS **inward**: move to a neighbour only if it's in-bounds, unvisited, and `heights[nbr] ≥ heights[cur]` (pass the current height as `prevHeight`; the guard rejects `heights[r][c] < prevHeight`).
4. Sweep every cell; if `pac[r][c] && atl[r][c]`, add `[r,c]` to the result.

> **Seed each border cell with its own height** so it always passes its first check — then it flows to equal-or-higher neighbours only.

## Code (clean canonical)
```cpp
class Solution {
public:
    vector<vector<int>> pacificAtlantic(vector<vector<int>>& heights) {
        int rows = heights.size(), cols = heights[0].size();
        vector<vector<bool>> pac(rows, vector<bool>(cols, false));
        vector<vector<bool>> atl(rows, vector<bool>(cols, false));

        function<void(int,int,vector<vector<bool>>&,int)> dfs =
            [&](int r, int c, vector<vector<bool>>& vis, int prevH) {
                if (r < 0 || c < 0 || r == rows || c == cols ||   // bounds FIRST
                    vis[r][c] || heights[r][c] < prevH) return;    // then visited/height
                vis[r][c] = true;
                dfs(r + 1, c, vis, heights[r][c]);
                dfs(r - 1, c, vis, heights[r][c]);
                dfs(r, c + 1, vis, heights[r][c]);
                dfs(r, c - 1, vis, heights[r][c]);
            };

        for (int c = 0; c < cols; ++c) {                 // top → Pacific, bottom → Atlantic
            dfs(0, c, pac, heights[0][c]);
            dfs(rows - 1, c, atl, heights[rows - 1][c]);
        }
        for (int r = 0; r < rows; ++r) {                 // left → Pacific, right → Atlantic
            dfs(r, 0, pac, heights[r][0]);
            dfs(r, cols - 1, atl, heights[r][cols - 1]);
        }

        vector<vector<int>> res;
        for (int r = 0; r < rows; ++r)
            for (int c = 0; c < cols; ++c)
                if (pac[r][c] && atl[r][c]) res.push_back({r, c});
        return res;
    }
};
```
**TC:** O(m·n) — each cell visited O(1) times per ocean (2 floods).  **SC:** O(m·n) — two visited grids + recursion stack.

## Other Approaches
- **Brute force — DFS/BFS from every cell to each ocean** *(the naive instinct)*: for each of `m·n` cells, flood outward and check if it can reach the Pacific edge and the Atlantic edge. **O((m·n)²)** — at 200×200 that's ~1.6e9, TLE. Correct but quadratic because it re-walks the same paths for every source.
- **Multi-source reverse DFS from the borders** *(the canonical block above — optimal)*: flip source and target, flood each ocean once, intersect. **O(m·n)** time / space. The standard interview answer.
- **Multi-source reverse BFS** *(same complexity, stack-safe)*: seed a queue with all Pacific border cells (then a second pass for Atlantic), expand to equal-or-higher unvisited neighbours. Identical O(m·n); prefer it if worried about deep recursion on a 200×200 monotone grid. Order doesn't matter here (no "levels"/distance to compute), so DFS vs BFS is purely a stack-safety choice.

**Family map:** parent grid flood-fill [[1-200-number-of-islands|Number of Islands]] / [[2-695-max-area-of-island|Max Area of Island]] (reachability). Closest sibling: [[4-130-surrounded-regions|Surrounded Regions]] — **same border-anchored inward flood** ("reachable from an edge = special"). Contrast [[3-994-rotting-oranges|Rotting Oranges]], which needs BFS **levels** because it measures *time*; here levels are irrelevant, so DFS is fine.

## Pitfalls
- **Reverse the flow** — flood **ocean → land** (uphill, height **≥**), not land → ocean per cell. Simulating from every source is the O((m·n)²) trap.
- **Invert the height test** — the guard is `heights[r][c] < prevHeight → return`, i.e. you *keep* going only to equal-or-higher cells. Using `>` (or `<=`) silently floods the wrong direction.
- **Bounds before index** — order the guard `r<0 || c<0 || r==rows || c==cols` **before** `vis[r][c]`/`heights[r][c]`, or you dereference out of range and crash.
- **Two separate visited grids** — a cell can drain to both oceans, so Pacific and Atlantic must not share one `visited`.
- **Seed each border with its own height** — pass `heights[border]` (not `0` or `INT_MIN` blindly); passing a too-low sentinel is fine only because every real height ≥ 0, but the cell's own height is the clear, correct seed.
- **DFS depth** — a monotone 200×200 grid can recurse ~40k deep; safe on LeetCode but switch to BFS/explicit stack if the platform's stack is tight.
