# 2. 695. Max Area of Island (LC 695) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/max-area-of-island/)

**Pattern:** Graph DFS/BFS — **grid flood-fill / connected components**, "return a size, not a count"
**Trigger:** "largest island / biggest connected region / max area of `1`s in a grid" → same scan-and-flood as [[1-200-number-of-islands|counting islands]], but each flood **returns the number of cells it filled** and you keep the running `max`

## Problem (1-liner)
Given an `m × n` binary matrix, an island = land cells (`1`) connected **4-directionally**. **Area** = count of `1`-cells in the island. Return the **maximum** island area; `0` if there's no land.
**Examples:** the classic scattered grid → `6` (not `11` — the big blob isn't fully connected) · all-water `[[0,…,0]]` → `0`.
**Constraints:** `1 ≤ m, n ≤ 50` (≤ 2500 cells — recursion depth is a non-issue here, unlike the 300×300 [[1-200-number-of-islands|#200]] grid).

## Idea (1-liner)
Identical scaffold to Number of Islands: scan every cell, and the first time you hit unvisited land, flood-fill its whole component. The **only** change is what the flood-fill hands back — instead of just marking cells visited, the DFS **returns the area** of the component (`1` for itself `+` the areas its neighbours return). The outer loop takes `max` over every island's returned area. "Count components" → "measure each component and keep the biggest."

## Approach
1. **Outer scan:** loop every cell; on unvisited land, call `area = dfs(...)` and `maxArea = max(maxArea, area)`.
2. **Flood-fill DFS:** mark the cell visited, seed `area = 1` (counts this cell), then for each in-bounds unvisited land neighbour `area += dfs(neighbour)`.
3. **Return `area`** up the recursion — every cell contributes exactly `1` because it's marked visited on entry and never re-entered.

> **Load-bearing insight:** seed `area = 1` for the *current* cell and **add the children's returns** — this is the "return info up" shape (cousin of tree post-order). Mark visited **on entry** so each cell is counted once; marking on pop would let a cell be summed from multiple neighbours.

## Code (clean canonical)
```cpp
class Solution {
    int dfs(vector<vector<int>>& grid, int r, int c) {
        int rows = grid.size(), cols = grid[0].size();
        if (r < 0 || r >= rows || c < 0 || c >= cols || grid[r][c] != 1) return 0;
        grid[r][c] = 0;                                    // sink => visited in-place
        return 1 + dfs(grid, r + 1, c) + dfs(grid, r - 1, c)
                 + dfs(grid, r, c + 1) + dfs(grid, r, c - 1);
    }
public:
    int maxAreaOfIsland(vector<vector<int>>& grid) {
        int rows = grid.size(), cols = grid[0].size(), maxArea = 0;
        for (int i = 0; i < rows; ++i)
            for (int j = 0; j < cols; ++j)
                if (grid[i][j] == 1) maxArea = max(maxArea, dfs(grid, i, j));
        return maxArea;
    }
};
```
**TC:** O(m·n) — each cell visited once.  **SC:** O(m·n) worst-case recursion stack; no `visited` grid since we sink to `0` in place. *Caveat: mutates the input.*

## My Solution
```cpp
class Solution {
    int dfs(vector<vector<int>>& grid, vector<vector<int>>& visited, int row, int col) {
        int area = 1, rows = grid.size(), cols = grid[0].size();
        visited[row][col] = 1;
        int drow[] = {0, 0, 1, -1};
        int dcol[] = {1, -1, 0, 0};
        for (int i = 0; i < 4; i++) {
            int newRow = row + drow[i], newCol = col + dcol[i];
            if (newRow >= 0 && newRow < rows && newCol >= 0 && newCol < cols &&
                !visited[newRow][newCol] && grid[newRow][newCol] == 1)
                area += dfs(grid, visited, newRow, newCol);
        }
        return area;
    }
public:
    int maxAreaOfIsland(vector<vector<int>>& grid) {
        int rows = grid.size(), cols = grid[0].size();
        vector<vector<int>> visited(rows, vector<int>(cols, 0));
        int maxArea = 0;
        for (int row = 0; row < rows; row++)
            for (int col = 0; col < cols; col++)
                if (!visited[row][col] && grid[row][col] == 1)
                    maxArea = max(maxArea, dfs(grid, visited, row, col));
        return maxArea;
    }
};
```
**Correct**, same O(m·n) time. `area = 1` base + adding children's returns is exactly right (no off-by-one), and it marks `visited` **on entry**. Only difference vs canonical: keeps a separate `visited` grid → O(m·n) **extra** space but **preserves the input**. Keep this whenever the grid must not be mutated; use the sink-in-place version when it can.

## Other Approaches
- **DFS, sink in place** *(optimal on space)* — the canonical block above. O(m·n) time, O(m·n) worst stack, no extra grid. Default when mutating is allowed.
- **DFS with `visited` grid** — your version. Same time, O(m·n) extra space, non-destructive.
- **BFS with a queue** — same O(m·n) time / O(min(m,n)) frontier; count cells as you pop them (mark on **push**). Iterative → no stack-overflow risk; reach for it on huge grids ([[1-200-number-of-islands|#200]]'s 300×300). Not needed at 50×50.
- **Union-Find (DSU)** — union adjacent land cells, then the answer is the **largest component size** (track size per root). O(m·n·α) ≈ linear. Overkill statically, but the tool when land is added incrementally and you need the running max area.

## Pitfalls
- **Return the size, don't just mark** — the whole delta from #200 is that DFS returns `1 + Σ children`; forgetting the `+1` or the accumulation gives wrong areas.
- **Mark visited on entry/push, not on pop** — else a cell is summed from multiple neighbours → inflated area.
- **`maxArea = 0` handles no-island** — seed at `0`, not `INT_MIN`; the empty-grid answer is `0`.
- **4 directions only** — diagonals don't connect (that's why Example 1 is `6`, not `11`).
- **Sink-in-place mutates the grid** — keep the `visited` array if the caller needs the grid intact.
- **Bounds before index** — order the guard so out-of-range short-circuits before `grid[nr][nc]`.
