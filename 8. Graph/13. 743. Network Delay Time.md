# 13. 743. Network Delay Time (LC 743) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/network-delay-time/)

**Pattern:** Weighted Graph — **single-source shortest path on non-negative weights** ⇒ **Dijkstra (lazy, min-heap)**
**Trigger:** "signal from node `k` reaches **all** nodes; return the **minimum time for the last one** / max over the shortest arrival times" → shortest path from one source, weights ≥ 0 ⇒ Dijkstra; the answer is the **max of all `dist[i]`** (the slowest node), or `-1` if any is unreachable.

## Problem (1-liner)
Directed weighted graph of `n` nodes (labeled `1..n`), edges `times[i] = (u, v, w)`. Send a signal from `k`; return the **minimum time for *all* `n` nodes to receive it** = the **maximum** shortest-path distance from `k`. If any node is unreachable, return `-1`.
**Examples:** `times=[[2,1,1],[2,3,1],[3,4,1]], n=4, k=2` → `2` (node 4 arrives last at t=2) · `[[1,2,1]], n=2, k=1` → `1` · `[[1,2,1]], n=2, k=2` → `-1` (node 1 unreachable).
**Constraints:** `1 ≤ k ≤ n ≤ 100`, `edges ≤ 6000`, `0 ≤ w ≤ 100`, no self-loops, unique `(u,v)`. **Edge:** weights can be `0`; some nodes may be unreachable → `-1`.

## Idea (1-liner)
"Time for all nodes" = time for the **slowest** node = `max(dist[1..n])` where `dist` is the shortest distance from `k`. Weights are non-negative ⇒ **Dijkstra**: pop the closest unfinished node from a min-heap, relax its out-edges. When the heap drains, every reachable node holds its final shortest time; unreachable ones stay `INT_MAX` ⇒ `-1`.

> **Load-bearing insight:** the answer is a **max, not a sum** — you want the last arrival, so take the maximum over all shortest distances. Any node still at `INT_MAX` means the network is disconnected from `k` ⇒ `-1`.

## Approach
1. **Build adjacency list** `adj[u] = {(v, w), …}` (directed).
2. **Init:** `dist` array `INT_MAX`; `dist[k] = 0`; min-heap seeded with `{0, k}` (ordered by distance).
3. **Relax:** pop `{d, node}` (smallest `d`). For each `(nbr, w)`: if `d + w < dist[nbr]`, update `dist[nbr]` and push `{d+w, nbr}`.
4. **Answer:** `ans = max(dist[1..n])`; if any `dist[i] == INT_MAX` → `-1`, else `ans`.

> **Lazy Dijkstra (no decrease-key):** a node can sit in the heap multiple times with stale distances. Optional guard `if (d > dist[node]) continue;` after popping skips outdated entries — not needed for correctness (a stale pop relaxes nothing new) but trims wasted work.

## Code (clean canonical)
```cpp
class Solution {
public:
    int networkDelayTime(vector<vector<int>>& times, int n, int k) {
        vector<vector<pair<int,int>>> adj(n + 1);           // adj[u] = {(v, w)}
        for (auto& t : times) adj[t[0]].push_back({t[1], t[2]});

        vector<int> dist(n + 1, INT_MAX);
        priority_queue<pair<int,int>, vector<pair<int,int>>,
                       greater<>> pq;                        // min-heap {dist, node}
        dist[k] = 0;
        pq.push({0, k});

        while (!pq.empty()) {
            auto [d, node] = pq.top(); pq.pop();
            if (d > dist[node]) continue;                    // skip stale entry
            for (auto& [nbr, w] : adj[node]) {
                if (d + w < dist[nbr]) {
                    dist[nbr] = d + w;
                    pq.push({d + w, nbr});
                }
            }
        }

        int ans = 0;
        for (int i = 1; i <= n; ++i) {
            if (dist[i] == INT_MAX) return -1;               // unreachable node
            ans = max(ans, dist[i]);
        }
        return ans;                                          // slowest arrival time
    }
};
```
**TC:** O(E log E) — each edge pushes at most one heap entry, heap ops are log-sized.  **SC:** O(V + E) — adjacency list + heap + dist.

## My Solution
Your solution is **correct and optimal** ✅ — classic lazy Dijkstra. Passes all three examples, handles `0`-weight edges (Dijkstra only needs weights ≥ 0), and unreachable nodes stay `INT_MAX` → `-1`. It works even **without** a stale-skip: a stale pop carries a larger `dis`, so every `dis + edgeW < distance[nbr]` fails and nothing bad happens — it just costs a few extra pops. Refinements: add `if (dis > distance[node]) continue;` (shows you know why lazy Dijkstra is correct), and fold the `-1` check into the max loop instead of seeding `time = INT_MIN`.

## Other Approaches
- **Bellman-Ford (brute relax)** — relax all `E` edges `V-1` times. **O(V·E)** / O(V). Simpler to code and the go-to if weights could be **negative** (Dijkstra breaks then). Fine here (`V=100, E=6000`), just slower.
- **Floyd-Warshall (all-pairs)** — `dist[i][j]` via `O(V³)`; answer = `max(dist[k][*])`. `100³ = 1M`, acceptable, but overkill for single-source.
- **Dijkstra with min-heap** *(optimal, canonical)* — the block above. O(E log E) / O(V+E). Default answer whenever weights are non-negative and you need one source.

## Pitfalls
- **Answer is a `max`, not a sum** — you want the *last* node to receive the signal, so take the maximum shortest distance.
- **`-1` on disconnection** — any node left at `INT_MAX` means unreachable from `k` ⇒ return `-1` (Example 3).
- **Overflow guard** — don't relax from an `INT_MAX` node (`INT_MAX + w` overflows). Seeding only `dist[k]=0` and pushing only relaxed nodes avoids ever popping an `INT_MAX` distance here.
- **Dijkstra needs weights ≥ 0** — with a negative edge, a finalized node could later get a shorter path; use Bellman-Ford instead.
- **Mark on relaxation, min-heap ordered by distance** — ordering by node id or marking visited on push (before it's finalized) breaks Dijkstra.
- **Nodes are 1-indexed** — size arrays `n+1` and loop `1..n`.
