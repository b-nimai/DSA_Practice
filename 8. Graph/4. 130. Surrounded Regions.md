# 4. 130. Surrounded Regions (LC 130) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/surrounded-regions/)

**Pattern:** Graph flood-fill — **border-anchored BFS/DFS** (invert the problem: flood from the *safe* boundary, not the interior)
**Trigger:** "capture regions **fully enclosed** by `X`; regions touching an **edge** survive" → don't hunt for enclosed regions directly — **flood every `O` reachable from the border, mark it safe, then flip the rest**

## Problem (1-liner)
Given an `m × n` board of `'X'`/`'O'`, flip **in-place** every `'O'` that belongs to a region **completely surrounded** by `'X'`. A region survives only if any of its `'O'`s sits on the **border** (row/col 0 or last).
**Examples:** `[[X,X,X,X],[X,O,O,X],[X,X,O,X],[X,O,X,X]]` → middle blob captured, bottom-left `O` (on edge) stays · `[[X]]` → `[[X]]` (nothing to do).
**Constraints:** `1 ≤ m, n ≤ 200`. Cells are only `'X'` or `'O'`.

## Idea (1-liner)
Detecting "is this region enclosed?" per-region is fiddly. **Invert it:** an `O` is *safe* iff it's connected to the **border**. So flood-fill from every border `O`, tag all reachable `O`s (temp mark `'#'`), then sweep once: untouched `O` → `X` (captured), tagged `'#'` → `O` (restored).

> **Load-bearing insight:** flood from the **boundary inward**, not the interior outward. "Reachable from an edge = safe" turns a hard enclosure test into a trivial reachability flood — the same trick powers [[5-417-pacific-atlantic-water-flow|Pacific Atlantic]].

## Approach
1. **Seed:** scan the 4 borders; push every border `'O'` into the queue, mark it `'#'` (safe/visited).
2. **BFS inward:** pop a cell, and for each 4-neighbour that's still `'O'`, mark `'#'` and push. Keep going until the queue drains — this tags the entire border-connected component.
3. **Final sweep:** for every cell — `'O'` (never reached) → `'X'` (capture); `'#'` (border-safe) → `'O'` (restore).

> **The temp marker `'#'` *is* the visited set** — because the guard only ever pushes cells that are still `'O'`, a marked cell is never re-enqueued. No separate `visited` grid needed.

## Code (clean canonical)
```cpp
class Solution {
public:
    void solve(vector<vector<char>>& board) {
        int rows = board.size(), cols = board[0].size();
        queue<pair<int,int>> q;

        auto seed = [&](int r, int c) {                  // push a border 'O', mark safe
            if (board[r][c] == 'O') { board[r][c] = '#'; q.push({r, c}); }
        };
        for (int i = 0; i < rows; ++i) { seed(i, 0); seed(i, cols - 1); }
        for (int j = 0; j < cols; ++j) { seed(0, j); seed(rows - 1, j); }

        int dr[] = {0, 0, 1, -1}, dc[] = {1, -1, 0, 0};
        while (!q.empty()) {
            auto [r, c] = q.front(); q.pop();
            for (int d = 0; d < 4; ++d) {
                int nr = r + dr[d], nc = c + dc[d];
                if (nr >= 0 && nr < rows && nc >= 0 && nc < cols && board[nr][nc] == 'O') {
                    board[nr][nc] = '#';                  // mark on discovery
                    q.push({nr, nc});
                }
            }
        }

        for (int i = 0; i < rows; ++i)
            for (int j = 0; j < cols; ++j) {
                if (board[i][j] == 'O') board[i][j] = 'X';       // enclosed → capture
                else if (board[i][j] == '#') board[i][j] = 'O';  // safe → restore
            }
    }
};
```
**TC:** O(m·n) — every cell visited O(1) times.  **SC:** O(m·n) worst case for the queue (board doubles as visited via `'#'`).

## My Solution (correct — with cleanup)
My original BFS was **correct** and passed. Two refinements only:
```cpp

...
for (int i = q.size(); i > 0; i--) { ... }                 // ❌ pointless level-drain
```

- **Drop `visited`** — the `board[nr][nc] == 'O'` guard + marking `'1'`/`'#'` already prevents re-enqueue. The grid *is* the visited set → saves an O(m·n) array.
- **Drop the level-order wrapper** — `for (int i = q.size(); ...)` serves no purpose here (we don't need BFS levels), and reusing `i` for both the drain and the direction index is a readability trap even though it works (inner `i` is block-scoped). Just pop one cell per iteration.

*(Marker `'1'` vs `'#'` is cosmetic — any sentinel that isn't `'O'`/`'X'` works.)*

## Other Approaches
- **Border DFS** *(optimal, most common in interviews)* — same invert-the-problem idea, recursion instead of a queue. From each border `'O'`, `dfs` marks `'#'` and recurses on 4-neighbours; then the same final sweep. O(m·n) time, **O(m·n) stack** worst case — on a 200×200 all-`O` board a snaking region can recurse ~40k deep and risk stack overflow, so **BFS is the safer pick at these constraints.**
- **Multi-source BFS from all borders** *(the canonical block above)* — seed every border `O` at once; cleanest and stack-safe.
- **Union-Find** — union all border `O`s to a virtual "safe" node and adjacent `O`s together; capture any `O` not connected to the safe node. O(m·n·α) — works but overkill; reach for it only if the problem adds dynamic connectivity.
- **Naive per-region enclosure test** *(brute / wrong instinct)* — flood each interior region and check if it touches an edge, flipping only if not. Doable but clumsier and easy to double-scan; the border-flood inversion is strictly simpler.

## Pitfalls
- **Invert the problem** — flood from the **border**, not the interior. Trying to detect enclosure directly is the classic overcomplication.
- **Mark on discovery** (when you push/enter), not on pop — else a cell reached by several neighbours is enqueued multiple times → re-work / blow-up.
- **The temp marker must differ from both `'O'` and `'X'`** — reusing `'X'` loses the ability to distinguish captured-vs-safe in the final sweep.
- **Restore before you finish** — the final pass must flip `'#'` back to `'O'`; forgetting it leaves stray sentinels on the board.
- **Bounds before index** — order the guard so out-of-range short-circuits before touching `board[nr][nc]`.
- **`1 × 1` / single row-col** — every cell is a border cell, so nothing is ever captured; the border-seed loop handles it for free.
- **DFS stack depth** — at `m,n ≤ 200` a pathological region can overflow the native stack; prefer BFS or an explicit stack.
