# Graph — Index

60-second whole-pattern recall. Read the **Trigger** column and try to recall the **Idea** before peeking.

**Core skeleton (grid flood-fill):** scan every cell; on an unvisited land cell `++count` and DFS/BFS-sink its whole component so it's never re-counted. **Mark visited on discovery, not on pop.** BFS/queue when recursion depth (up to `m·n`) could overflow; Union-Find when components merge dynamically.

| 14 | [Cheapest Flights Within K Stops](https://leetcode.com/problems/cheapest-flights-within-k-stops/) | 787 | M | "cheapest `src → dst` with **at most `k` stops**" | **Hop-limited shortest path ⇒ BFS on a `(node, stops)` state, plain FIFO `queue` — Dijkstra breaks here.** Push `{cost, stops, node}`, pop from the **front**, `if (stops > k) continue;`, relax `cost + w < distance[nbr]` → update + push `{.., stops+1, ..}`. `k` stops = **`k+1` edges**; the guard goes at **pop, not push** (a `stops == k+1` entry is a valid *answer*, it just can't fly again). | 15 | [Min Cost to Connect All Points](https://leetcode.com/problems/min-cost-to-connect-all-points/) | 1584 | M | "connect **all** points at **minimum total cost**", "exactly one simple path between any two" (= a tree) | **Connect-everything-cheapest ⇒ MST (Prim), not Dijkstra — there is no `src`/`dst`.** Edges aren't given: the graph is **complete and implicit**, node = **index**, weight `(i,j)` = Manhattan `|xi-xj|+|yi-yj|`. Prim: min-heap of `{weight, node}`, pop → `if (visited) continue;` → mark → `total += w` → push unvisited neighbours. **Push `w`, NOT `dis + w` — that one token is Prim vs Dijkstra** (cost to *attach* a node, not to *reach* it). Heap+adjList is O(n² log n)/O(n²); since the graph is complete, **dense Prim** (`minDist[]`, weights on the fly, no graph at all) is O(n²)/O(n) and measured 86ms → 5ms at n=1000. `n=1` → `0` for free. Alt: Kruskal + DSU, sort all n²/2 edges, stop after n-1 unions | O(n²) | O(n) |

## When to reach for Graph
1. **Grid / matrix** with 4- or 8-directional movement → flood-fill (islands, regions, rotting), often BFS for shortest/level.
2. **Connected components / bipartite / cycle** detection → DFS or Union-Find.
3. **Dependencies / ordering / "course schedule"** → topological sort (Kahn's BFS or DFS post-order).
4. **Shortest path in an *unweighted* graph** → plain BFS; weighted → Dijkstra/Bellman-Ford (breadth block).

## Common Pitfalls Across the Pattern
- **Mark visited on discovery** (enqueue/enter), not when you dequeue — the classic re-count / blow-up bug.
- **Bounds before index** — order the guard so out-of-range short-circuits before `grid[nr][nc]`.
- **Recursion depth** — DFS on large grids/graphs can overflow the native stack; offer BFS or an explicit stack.
- **Chars vs ints in grids** — compare against `'1'`/`'0'`, not `1`/`0`.
