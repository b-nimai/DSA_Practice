# Graph — Index

60-second whole-pattern recall. Read the **Trigger** column and try to recall the **Idea** before peeking.

**Core skeleton (grid flood-fill):** scan every cell; on an unvisited land cell `++count` and DFS/BFS-sink its whole component so it's never re-counted. **Mark visited on discovery, not on pop.** BFS/queue when recursion depth (up to `m·n`) could overflow; Union-Find when components merge dynamically.

| 1 | [Number of Islands](https://leetcode.com/problems/number-of-islands/) | 200 | M | "count connected blobs/regions of `1`s in a grid" | Grid = implicit graph, **#islands = #connected components**. Scan cells; each unvisited land cell is a fresh component → `++count`, then flood-fill (DFS sink to `'0'`, or BFS) every 4-adjacent land cell so the rest of that island is absorbed. **Mark visited on entry/push**, never on pop (else double-count/TLE). 4 dirs only — diagonals don't connect. 300×300 all-land ⇒ 90k-deep DFS can blow the stack → prefer BFS. Dynamic-add variant → Union-Find (305) | O(m·n) | O(m·n) |
| 2 | [Max Area of Island](https://leetcode.com/problems/max-area-of-island/) | 695 | M | "largest island / biggest connected region / max area of `1`s" | **Same scaffold as 200, but the flood returns a *size*.** DFS seeds `area = 1` for the current cell and **adds children's returns** (`1 + Σ dfs(neighbour)`); outer loop keeps `max` over every island. "Count components" → "measure each, keep the biggest." Mark visited on entry so each cell counts once; seed `maxArea = 0` (no-island answer). Sink-in-place = O(1) aux; `visited` grid = non-destructive. 50×50 so depth safe | O(m·n) | O(m·n) |
| 3 | [Rotting Oranges](https://leetcode.com/problems/rotting-oranges/) | 994 | M | "every minute all rotten cells infect neighbours **simultaneously**; min minutes till none fresh" | **NOT DFS — multi-source BFS, level = time.** Seed queue with **all** rotten cells (minute-0 frontier) + count `fresh`; expand **ring by ring**, each drained ring = 1 minute. Freeze `sz = q.size()` before draining so this minute's cells don't mix with next. ## When to reach for Graph
1. **Grid / matrix** with 4- or 8-directional movement → flood-fill (islands, regions, rotting), often BFS for shortest/level.
2. **Connected components / bipartite / cycle** detection → DFS or Union-Find.
3. **Dependencies / ordering / "course schedule"** → topological sort (Kahn's BFS or DFS post-order).
4. **Shortest path in an *unweighted* graph** → plain BFS; weighted → Dijkstra/Bellman-Ford (breadth block).

## Common Pitfalls Across the Pattern
- **Mark visited on discovery** (enqueue/enter), not when you dequeue — the classic re-count / blow-up bug.
- **Bounds before index** — order the guard so out-of-range short-circuits before `grid[nr][nc]`.
- **Recursion depth** — DFS on large grids/graphs can overflow the native stack; offer BFS or an explicit stack.
- **Chars vs ints in grids** — compare against `'1'`/`'0'`, not `1`/`0`.
