# Graph — Index

60-second whole-pattern recall. Read the **Trigger** column and try to recall the **Idea** before peeking.

**Core skeleton (grid flood-fill):** scan every cell; on an unvisited land cell `++count` and DFS/BFS-sink its whole component so it's never re-counted. **Mark visited on discovery, not on pop.** BFS/queue when recursion depth (up to `m·n`) could overflow; Union-Find when components merge dynamically.

"Count components" → "measure each, keep the biggest." Mark visited on entry so each cell counts once; seed `maxArea = 0` (no-island answer). | 6 | [Course Schedule](https://leetcode.com/problems/course-schedule/) | 207 | M | "given **prerequisites / dependencies**, can you finish all / is there a valid order?" | **Topological sort = cycle detection (Kahn's BFS).** Edge `b → a` for each `[a,b]` (b before a); `indegree[a]++`. Seed queue with all **indegree-0** nodes (no prereqs), peel one at a time (`count++`), decrement each dependent's indegree, push when it hits 0. `count == numCourses` ⇒ acyclic ⇒ `true`; a cycle never reaches indegree 0 so it's left out. You need the *count*, not the order. Alt: DFS 3-color, in-progress neighbour = back edge = cycle | O(V+E) | O(V+E) |
| 7 | [Course Schedule II](https://leetcode.com/problems/course-schedule-ii/) | 210 | M | "output **a valid order** to do everything (empty if impossible)" | **Same `b → a` DAG as 207 — now emit the order.** **Kahn's BFS:** append each node **on pop** (`topo.push_back` when dequeued) — the pop sequence *is* the topo order; `topo.size() != numCourses` ⇒ cycle ⇒ `{}`. **DFS alt:** 3-color, collect **post-order** (push on finish), then **reverse** it (prereqs finish first); back edge (in-progress neighbour) ⇒ cycle ⇒ `{}`. | 8 | [Is Graph Bipartite?](https://leetcode.com/problems/is-graph-bipartite/) | 785 | M | "split nodes into **two groups** so **every edge crosses** between them (no edge inside a group)" | **Bipartite ⇔ 2-colorable ⇔ no odd cycle.** Color a start node `0`, force each neighbour to the **opposite** color (`!color`); a neighbour **already carrying your color** = odd cycle ⇒ `false`. `colors` init `-1` (uncolored ≠ color). **Loop over all nodes** — graph is **disconnected**, seed each component. ## When to reach for Graph
1. **Grid / matrix** with 4- or 8-directional movement → flood-fill (islands, regions, rotting), often BFS for shortest/level.
2. **Connected components / bipartite / cycle** detection → DFS or Union-Find.
3. **Dependencies / ordering / "course schedule"** → topological sort (Kahn's BFS or DFS post-order).
4. **Shortest path in an *unweighted* graph** → plain BFS; weighted → Dijkstra/Bellman-Ford (breadth block).

## Common Pitfalls Across the Pattern
- **Mark visited on discovery** (enqueue/enter), not when you dequeue — the classic re-count / blow-up bug.
- **Bounds before index** — order the guard so out-of-range short-circuits before `grid[nr][nc]`.
- **Recursion depth** — DFS on large grids/graphs can overflow the native stack; offer BFS or an explicit stack.
- **Chars vs ints in grids** — compare against `'1'`/`'0'`, not `1`/`0`.
