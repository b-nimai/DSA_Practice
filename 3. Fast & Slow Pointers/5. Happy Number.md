# 5. Happy Number (LC 202) — Easy
🔗 [Open on LeetCode](https://leetcode.com/problems/happy-number/)

**Pattern:** Fast & Slow Pointers — Floyd's on a function iteration `x → f(x)` (no linked list)
**Trigger:** "repeat a transform until it reaches 1 **or loops forever**" → the sequence either hits a fixed point or cycles → detect the cycle with slow/fast, O(1) space (no hash set)

## Problem (1-liner)
Repeatedly replace `n` with the sum of the squares of its digits; return `true` if this reaches `1`, `false` if it loops forever.
**Examples:** `19` → `1` (`82→68→100→1`) ⇒ `true` · `2` → cycles ⇒ `false`

## Idea (1-liner)
Define `f(x)` = sum of squares of digits. The sequence `n, f(n), f(f(n)), …` is a linked list of numbers, so an endless loop **is** a cycle. Floyd's tortoise & hare detects it: happy numbers converge to the fixed point `1` (`f(1)=1`), unhappy ones meet inside a non-1 cycle.

## Approach
1. Helper `f(x)`: while `x`, peel `x % 10`, add its square, `x /= 10`.
2. `slow = fast = n`. Advance `slow = f(slow)` and `fast = f(f(fast))` until they meet.
3. They always meet (finite state space ⇒ the sequence is eventually periodic). If the meeting value is `1`, it's happy; otherwise it's a non-1 cycle.

**Why no hash set:** the sequence is eventually periodic, so it behaves exactly like a linked list with a cycle — Floyd's gives O(1) space instead of the O(n) a `seen` set would cost.

**Why `int` is safe:** max input ~2.1×10⁹ has 10 digits, so the largest possible `f(x)` is `9²·10 = 810` — the sequence collapses below 1000 after one step. No overflow.

## Code (clean canonical)
```cpp
class Solution {
    int squareDigitSum(int x) {
        int sum = 0;
        while (x) {
            int d = x % 10;
            sum += d * d;
            x /= 10;
        }
        return sum;
    }
public:
    bool isHappy(int n) {
        int slow = n, fast = n;
        do {
            slow = squareDigitSum(slow);
            fast = squareDigitSum(squareDigitSum(fast));
        } while (slow != fast);
        return slow == 1;   // met at 1 ⇒ happy; met elsewhere ⇒ non-1 cycle
    }
};
```
**TC:** O(log n) per step, sequence length bounded by the (small) cycle ⇒ effectively O(log n)  **SC:** O(1)

## My Solution
```cpp
class Solution {
  int sumOfSquareOfDigits(int num) {
    int sum = 0;
    while(num) {
      int digit = num % 10;
      num /= 10;
      sum += digit * digit;
    }
    return sum;
  }
public:
    bool isHappy(int n) {
      int slow = n, fast = n;
      do {
        slow = sumOfSquareOfDigits(slow);
        fast = sumOfSquareOfDigits(sumOfSquareOfDigits(fast));
      } while(slow != fast);
      return fast == 1;
    }
};
```

> **Clean, correct, O(1) space** — this is the canonical `do/while` Floyd's-on-`f(x)`. Both pointers always meet (happy ⇒ at the fixed point `1`, unhappy ⇒ inside a non-1 cycle), and the single `return fast == 1` (`== slow` at exit) decides everything. The `do/while` advances before comparing, sidestepping the spurious-start-state trap. Nothing to refine.

## Other Approaches
- **Hash set of seen values** — store each value; if you revisit one, it's a cycle (and not 1). O(log n) time but O(k) space for the cycle. Correct and arguably more intuitive, but the interviewer will push for Floyd's O(1) space.
- **Hardcode the known cycle** — every unhappy number eventually hits `4` (the entry of the only non-1 cycle: `4→16→37→58→89→145→42→20→4`). So `while (n != 1 && n != 4) n = f(n); return n == 1;`. Slick, but relies on a memorized fact rather than a general technique.

## Pitfalls
- **It's Floyd's, not a linked list.** The "list" is the implicit sequence `x → f(x)`; recognizing that is the whole trick. Same template powers [Find the Duplicate Number](https://leetcode.com/problems/find-the-duplicate-number/) (LC 287) where the array forms `x → nums[x]`.
- **Don't compare before advancing.** Both pointers start at `n`; an up-front `slow == fast` check fires spuriously. The `do/while` advances first — same gotcha as [1. Linked List Cycle](1.%20Linked%20List%20Cycle.md).
- **Termination is guaranteed:** the value space is finite (collapses below 1000), so the sequence must eventually repeat — the loop can't run forever.
- Returning `true` for `n == 1`: handled — `do/while` computes `f(1)=1`, meets immediately, returns `1 == 1`. ✓
