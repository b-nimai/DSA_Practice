## Variants of Fast & Slow Pointers (cluster the problems)

| Variant | Problems | Signature |
|---------|----------|-----------|
| **Cycle existence** (Floyd's, phase 1 only) | 1 | `slow+=1`, `fast+=2` from `head`; meet ⇒ cycle; `fast` hits null ⇒ none |
| **Cycle entry** (Floyd's, phase 1 + 2) | 2 | after meet, reset one pointer to `head`; advance both by 1 ⇒ next meet = cycle start (proof: `head→entry = meet→entry` mod cycle) |
| **Middle / k-from-end** | 3, 6 | speed-ratio trick: `fast` runs at 2× ⇒ when `fast` reaches end, `slow` is at middle (2nd middle on even; start `fast = head->next` for 1st); or pre-advance `fast` by `k` then walk together ⇒ `slow` lands on k-from-end (use a dummy + gap `k+1` when you must *delete*, so `slow` stops on the predecessor) |
| **Find middle + reverse half** | 4, 7 | locate midpoint with slow/fast, reverse the back half in place, then compare/merge the two halves; restore by reversing back. Backbone of palindrome-check (compare) and reorder-list (interleave) |
| **Cycle in a function iteration** (non-list) | 5 | same Floyd's on `x → f(x)` (no pointers, just values); `do/while` until `slow == fast`, then test the meeting value. Used for "happy number" (`f` = digit-square-sum), "find the duplicate" (`f(x) = nums[x]`) |

## When to reach for Fast & Slow Pointers
1. Linked-list problem asking about a **cycle** (exists / entry node / length).
2. Need the **middle** or **k-th from the end** in one pass, O(1) extra space.
3. Any iteration `x → f(x)` where you suspect it eventually repeats and you want to detect or locate the cycle without a hash set.
4. Keywords: "cycle", "loop", "middle", "from the end", "happy number", "duplicate number" (cyclic-array reading).

## Common Pitfalls Across the Pattern
- **Loop guard:** always `fast && fast->next` before doing `fast = fast->next->next`. Checking only `fast` will crash on even-length lists.
- **Compare after moving**, not before — both pointers start at `head`, so an up-front `slow == fast` check fires spuriously.
- For cycle-entry (Floyd's phase 2): reset to `head` *after* the meet point, then both walk **at speed 1** — not 2. Easy to forget.
- For middle-of-list: parity matters — `[1,2,3,4]` returns the second middle (`3`) with the standard template; if you need the first middle, advance `fast` one step before the loop.
- Hash-set approach is correct but O(n) space — interviewer will push for Floyd's; learn it as the default.
- Don't mutate the list (sentinel marking) to detect cycles — destructive, breaks the input.

## C++ Fast & Slow Pointers templates
```cpp
// Cycle existence (Floyd's, phase 1)
bool hasCycle(ListNode* head) {
    ListNode *slow = head, *fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) return true;
    }
    return false;
}

// Cycle entry (Floyd's, phase 1 + 2)
ListNode* detectCycle(ListNode* head) {
    ListNode *slow = head, *fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) {                  // meet inside cycle
            ListNode* p = head;
            while (p != slow) { p = p->next; slow = slow->next; }
            return p;                        // cycle start
        }
    }
    return nullptr;
}

// Middle of linked list (returns 2nd middle on even length)
ListNode* middleNode(ListNode* head) {
    ListNode *slow = head, *fast = head;
    while (fast && fast->next) { slow = slow->next; fast = fast->next->next; }
    return slow;
}

// k-th from end (k ≥ 1, valid k)
ListNode* kthFromEnd(ListNode* head, int k) {
    ListNode *slow = head, *fast = head;
    for (int i = 0; i < k; i++) fast = fast->next;   // pre-advance
    while (fast) { slow = slow->next; fast = fast->next; }
    return slow;
}
```
