# 10. Circular Array Loop (LC 457) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/circular-array-loop/)

**Pattern:** Fast & Slow Pointers — Floyd's on the **index-movement function** `next(i) = ((i + nums[i]) % n + n) % n` (an array read as a functional graph, like LC 287 / LC 202), with **two extra acceptance rules**
**Trigger:** "circular array, each value is a signed **jump length**, find a cycle that is **all one direction** and **length > 1**" → run Floyd's on `next(i)`, but **reject a sign flip** (crossing forward/backward) and **reject a self-loop** (`next(i) == i`)

## Problem (1-liner)
`nums` holds non-zero ints; at index `i` you move `nums[i]` steps (forward if positive, backward if negative) with wraparound. Return `true` iff some cycle exists whose elements are **all positive or all negative** and whose length **`k > 1`**.
**Examples:** `[2,-1,1,2,2]` → `true` (`0→2→3→0`, all forward) · `[-1,-2,-3,-4,-5,6]` → `false` (only a size-1 loop) · `[1,-1,5,1,4]` → `true` (`3→4→3`)

## Idea (1-liner)
Read each slot as a pointer `i → next(i)`; pigeonhole on a finite index set means any walk eventually cycles, so use Floyd's tortoise & hare to detect a meet — but layer in the two problem-specific guards (same sign every hop, and `next(i) != i`) so the cycle you detect is a *valid* one. Mark exhausted paths with `0` to keep it O(n).

## Approach
1. **Index move helper:** `next(i) = ((i + nums[i]) % n + n) % n`. The double `% n` (`(...%n + n)%n`) is mandatory in C++ so negative jumps wrap correctly.
2. **For each start `i`** (skip if `nums[i] == 0`, our visited marker): set `slow = fast = i`.
3. **Advance with guards.** Before every hop check the **sign** of the slot you leave vs. the slot you land on: `nums[a] * nums[next(a)] > 0` means same direction. `fast` takes two hops, so validate **both**. The moment any product is `≤ 0`, the direction flipped — abandon this start.
4. **Meet ⇒ candidate cycle.** When `slow == fast`, verify **length > 1**: if `slow == next(slow)` it's a self-loop (`k == 1`) → invalid, break. Otherwise `return true`.
5. **Mark the dead path.** If the start yields no valid cycle, walk from `i` while the direction holds and set each visited slot to `0`. This guarantees every index is explored O(1) times overall ⇒ **O(n) total**.

**Why the sign guard works locally, not globally:** because we abandon the instant a sign flips, every index along any path we *keep* already shares a direction — so checking adjacent hops is enough; we never need to re-scan the whole loop.

**Why `next(i) == i` is the length-1 test:** a single element pointing at itself is exactly a cycle of length 1, which the problem forbids. (Happens when `nums[i]` is a multiple of `n`, e.g. `6` at index 5 with `n = 6`.)

## Code (clean canonical)
```cpp
class Solution {
public:
    bool circularArrayLoop(vector<int>& nums) {
        int n = nums.size();
        auto next = [&](int i) { return ((i + nums[i]) % n + n) % n; };

        for (int i = 0; i < n; i++) {
            if (nums[i] == 0) continue;                  // visited marker

            int slow = i, fast = i;
            // keep moving only while every hop preserves direction (same sign)
            while (nums[slow] * nums[next(slow)] > 0 &&
                   nums[fast] * nums[next(fast)] > 0 &&
                   nums[next(fast)] * nums[next(next(fast))] > 0) {
                slow = next(slow);
                fast = next(next(fast));
                if (slow == fast) {
                    if (slow == next(slow)) break;       // length-1 self loop → invalid
                    return true;                         // valid cycle, k > 1
                }
            }

            // dead path: zero out every slot reachable in this direction
            int j = i, dir = nums[i];
            while (nums[j] * dir > 0) {
                int nxt = next(j);
                nums[j] = 0;
                j = nxt;
            }
        }
        return false;
    }
};
```
**TC:** O(n) — each index is walked a constant number of times before being zeroed.  **SC:** O(1) — in-place marking, no extra structures.

## Other Approaches
- **Visited-set per start (no marking)** — for each `i`, walk with a `unordered_set` of indices on the current path; report a cycle when you revisit an index *and* the path length > 1, abort on sign change. Simple to reason about but O(n) extra space and easy to make O(n²) without the global `0` marking. The marking trick is what earns the O(1)-space follow-up.
- **Color / state array (white-gray-black DFS)** — treat it as cycle detection in a functional graph with a 3-state visited array. Correct, but O(n) space and more bookkeeping than Floyd's + marking.

## Pitfalls
- **Negative modulo:** plain `(i + nums[i]) % n` can be negative in C++. Always `((i + nums[i]) % n + n) % n`.
- **Validate both of `fast`'s hops** — `fast` moves twice per iteration, so the sign guard needs *three* products (slow's hop + fast's two hops), not one. Checking only the final landing lets a mid-step direction flip slip through.
- **The length-1 check is mandatory** — pure Floyd's would wrongly accept a self-loop. Test `slow == next(slow)` at the meet (see Example 2's lone size-1 cycle).
- **Don't forget the visited marking** — without zeroing dead paths you re-explore indices and the solution degrades to O(n²); the marking is the whole point of the O(n)/O(1) follow-up.
- **Same family, different `f`** — the "array as a function" framing is shared with [Find the Duplicate Number](https://leetcode.com/problems/find-the-duplicate-number/) (LC 287, `f(x)=nums[x]`) and [Happy Number](https://leetcode.com/problems/happy-number/) (LC 202, `f`=digit-square-sum). Here `f(i)=next(i)` plus the direction/length rules.
