# 1. Maximum Average Subarray I (LC 643) — Easy
🔗 [Open on LeetCode](https://leetcode.com/problems/maximum-average-subarray-i/)

**Pattern:** Sliding Window — fixed size `k` (slide, don't regrow)
**Trigger:** "subarray of length exactly k" + max/min of some aggregate → fixed window

## Problem (1-liner)
Find the contiguous subarray of length exactly `k` with the **maximum average**; return that average (error < 1e-5 OK).
**Example:** `nums = [1,12,-5,-6,50,3], k = 4` → `12.75` (`(12-5-6+50)/4`)

## Idea (1-liner)
Average is monotonic in the window sum (k is fixed), so track the max **sum**: build the first window, then slide — add the entering element, drop the leaving one — and divide once at the end.

## Approach
1. `sum = nums[0..k-1]`; `maxSum = sum`.
2. For `i = k .. n-1`: `sum += nums[i] - nums[i-k]`; `maxSum = max(maxSum, sum)`.
3. Return `(double) maxSum / k`.

## Code (clean canonical)
```cpp
double findMaxAverage(vector<int>& nums, int k) {
    int sum = 0;
    for (int i = 0; i < k; i++) sum += nums[i];
    int maxSum = sum;
    for (int i = k; i < (int)nums.size(); i++) {
        sum += nums[i] - nums[i - k];
        maxSum = max(maxSum, sum);
    }
    return (double) maxSum / k;
}
```
**TC:** O(n)  **SC:** O(1)

## My Solution
```cpp
class Solution {
public:
    double findMaxAverage(vector<int>& nums, int k) {
        int sum = 0;
        for(int i = 0; i < k; i++) sum += nums[i];

        int maxSum = sum, left = 0;
        for(int i = k; i < nums.size(); i++) {
          sum += nums[i] - nums[left++];
          maxSum = max(sum, maxSum);
        }

        return (double) maxSum / k;
    }
};
```

> **First cut:** divided every iteration and compared `avg` values with an `INT_MIN/2` sentinel. Works, but the division is wasted work and adds floating-point rounding inside the loop. Since `k` is fixed, comparing sums is equivalent — divide exactly once at the end. The sentinel isn't needed either: `n ≥ k ≥ 1`, so the first window always exists → seed `maxSum` with it.

## Other Approaches
- **Brute force:** sum every length-`k` window from scratch → O(n·k) / O(1).
- **Prefix sums:** `pre[i+1]-pre[i+1-k]` for each window → O(n) / O(n) — same cost, more memory than the slide.

## Pitfalls
- `nums.size()` is unsigned — cast to `int` before `i - k` (or loop differently) to avoid underflow surprises.
- Sum fits in 32-bit `int` here (`1e5 * 1e4 = 1e9`); if the bounds were larger use `long long`.
- Return type is `double` — cast before dividing, else integer division.
- "exactly k" → fixed window; don't reach for the grow/shrink (variable) template (that's [2. Minimum Size Subarray Sum](2.%20Minimum%20Size%20Subarray%20Sum.md)).
