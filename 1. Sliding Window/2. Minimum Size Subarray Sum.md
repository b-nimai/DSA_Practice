# 2. Minimum Size Subarray Sum (LC 209) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/minimum-size-subarray-sum/)

**Pattern:** Sliding Window — variable size (grow `right`, shrink `left` while still valid)
**Trigger:** "shortest subarray whose sum ≥ target", positive numbers → grow/shrink window

## Problem (1-liner)
Array of **positive** ints; return the **minimal length** of a contiguous subarray with sum `≥ target`, or `0` if none exists.
**Example:** `target = 7, nums = [2,3,1,2,4,3]` → `2` (`[4,3]`)

## Idea (1-liner)
Expand `right` adding to `sum`; whenever `sum ≥ target`, the window is valid — record its length and shrink from `left` (it can only get shorter), repeating until it's no longer valid.

## Approach
1. `left = 0`, `sum = 0`, `minLen = n + 1` (impossible sentinel).
2. For `right = 0 .. n-1`:
   - `sum += nums[right]`.
   - While `sum >= target`: `minLen = min(minLen, right - left + 1)`; `sum -= nums[left++]`.
3. Return `minLen == n + 1 ? 0 : minLen`.

## Code (clean canonical)
```cpp
int minSubArrayLen(int target, vector<int>& nums) {
    int n = nums.size();
    int minLen = n + 1, left = 0, sum = 0;
    for (int right = 0; right < n; right++) {
        sum += nums[right];
        while (sum >= target) {
            minLen = min(minLen, right - left + 1);
            sum -= nums[left++];
        }
    }
    return minLen == n + 1 ? 0 : minLen;
}
```
**TC:** O(n) (each index enters and leaves the window once)  **SC:** O(1)

## My Solution
```cpp
class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
      int size = nums.size();
      int minLen = size + 1, left = 0, sum = 0;
      for(int right = 0; right < size; right++) {
        sum += nums[right];
        while(sum >= target) {
          minLen = min(minLen, right - left + 1);
          sum -= nums[left++];
        }
      }
      return minLen == size + 1 ? 0 : minLen;
    }
};
```

## Other Approaches
- **Brute force:** every `(i, j)` pair, sum and check → O(n²) / O(1) (or O(n³) without a running sum).
- **Prefix sum + binary search:** `pre[]` is increasing (positives), so for each `i` binary-search the smallest `j` with `pre[j] - pre[i] ≥ target` → O(n log n) / O(n). Needed if numbers could be negative... except then the window trick breaks too — that becomes a different problem (deque / other).

## Pitfalls
- `while` (not `if`) on the shrink — keep contracting as long as the window stays valid.
- Sentinel must be unreachable: `n + 1` works since no subarray is longer than `n`. Don't return it raw — map back to `0`.
- Relies on all `nums[i] > 0` (removing from the left strictly lowers the sum). With zeros/negatives this approach is invalid.
- Record the length **before** the shrink step that breaks validity — i.e. inside the loop, while `sum >= target` still holds.
- `sum` fits in 32-bit `int` (`1e5 * 1e4 = 1e9 < 2.1e9`); `target` can be `1e9`, also fine. Larger bounds → `long long`.
- Contrast with [1. Maximum Average Subarray I](1.%20Maximum%20Average%20Subarray%20I.md): there the width is given (fixed window); here we're searching for the width (variable window).
