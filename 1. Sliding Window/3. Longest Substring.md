# 3. Longest Substring Without Repeating Characters (LC 3) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/longest-substring-without-repeating-characters/)

**Pattern:** Sliding Window — variable size (grow `right`, shrink `left` while *invalid*)
**Trigger:** "longest substring with no duplicates" → window stays valid while every char inside is unique

## Problem (1-liner)
Given a string `s`, return the **length of the longest substring** with no repeating characters.
**Example:** `s = "abcabcbb"` → `3` (`"abc"`); `s = "pwwkew"` → `3` (`"wke"`).

## Idea (1-liner)
Maintain a window `[left, right]` of distinct characters. Each step, extend `right`; if the new char is already in the window, shrink from `left` until it isn't — then update the answer.

## Approach
1. `left = 0`, `maxLen = 0`, frequency map `cnt`.
2. For `right = 0 .. n-1`:
   - `cnt[s[right]]++`.
   - While `cnt[s[right]] > 1` (window invalid — duplicate exists): `cnt[s[left++]]--`.
   - `maxLen = max(maxLen, right - left + 1)`.
3. Return `maxLen`.

## Code (clean canonical)
```cpp
int lengthOfLongestSubstring(string s) {
    int n = s.size(), left = 0, maxLen = 0;
    unordered_map<char, int> cnt;
    for (int right = 0; right < n; right++) {
        cnt[s[right]]++;
        while (cnt[s[right]] > 1) cnt[s[left++]]--;
        maxLen = max(maxLen, right - left + 1);
    }
    return maxLen;
}
```
**TC:** O(n) (each index enters and leaves the window once)  **SC:** O(min(n, Σ)) where Σ is the alphabet size

## My Solution
```cpp
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
      unordered_map<char, int>mpp;
      int left = 0, maxLen = 0;
      for(int right = 0; right < s.size(); right++) {
        mpp[s[right]]++;
        while(mpp[s[right]] > 1) {
          mpp[s[left]]--;
          left++;
        }
        maxLen = max(maxLen, right - left + 1);
      }
      return maxLen;
    }
};
```

## Other Approaches
- **Brute force:** every `(i, j)` pair, check uniqueness with a set → O(n³) / O(min(n, Σ)).
- **Last-seen index map (single pass, no inner loop):** store the most recent index of each char; when `s[right]` was seen at index `p ≥ left`, jump `left = p + 1`. Same O(n) / O(Σ) but `left` never revisits a position — one tighter pass.
  ```cpp
  int lengthOfLongestSubstring(string s) {
      unordered_map<char, int> last;
      int left = 0, maxLen = 0;
      for (int right = 0; right < (int)s.size(); right++) {
          auto it = last.find(s[right]);
          if (it != last.end() && it->second >= left) left = it->second + 1;
          last[s[right]] = right;
          maxLen = max(maxLen, right - left + 1);
      }
      return maxLen;
  }
  ```
- **Fixed array instead of hash map:** problem says ASCII-ish (letters, digits, symbols, spaces) → `int cnt[128] = {0};` is faster and avoids hashing overhead.

## Pitfalls
- `while` (not `if`) on the shrink — the duplicate could remain after one decrement only if `s[left] != s[right]`; keep shrinking until the count of `s[right]` is back to 1.
- Only `s[right]`'s count can become > 1 (every other char was already unique pre-insert), so checking *that one entry* is enough — don't loop over the whole map.
- In the last-seen variant, **guard `last[c] >= left`**: a stale index from before the current window must not pull `left` backward.
- `s.size()` is unsigned (`size_t`); comparing with `int right` warns. Use `int n = s.size();` or cast.
- "Substring" = contiguous; not subsequence. `"pwke"` from `"pwwkew"` is a subsequence, not a valid answer.
- Empty string → returns `0` naturally (loop doesn't execute).
- Contrast with [2. Minimum Size Subarray Sum](2.%20Minimum%20Size%20Subarray%20Sum.md): there we shrink **while valid** (looking for shortest); here we shrink **while invalid** (looking for longest) and record *after* restoring validity.
