# 2. Linked List Cycle II (LC 142) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/linked-list-cycle-ii/)

**Pattern:** Fast & Slow Pointers — Floyd's Tortoise & Hare (phase 1 + phase 2)
**Trigger:** "find the **node where the cycle begins**" (not just whether one exists) → Floyd's two-phase

## Problem (1-liner)
Given the head of a singly linked list, return the node where the cycle starts; return `nullptr` if there is no cycle. **Do not modify** the list.
**Example:** `head = [3,2,0,-4], pos = 1` → node at index 1 (the `2`).

## Idea (1-liner)
Phase 1 detects a cycle (like [1. Linked List Cycle](1.%20Linked%20List%20Cycle.md)); phase 2 exploits the math `head→entry ≡ meet→entry (mod L)` — reset one pointer to head, walk both at **speed 1**, they collide at the entry.

## Approach
1. **Phase 1:** `slow = fast = head`. Loop while `fast && fast->next`: `slow += 1`, `fast += 2`. If `slow == fast` → cycle exists, go to phase 2. If `fast` falls off → return `nullptr`.
2. **Phase 2:** `p = head`. While `p != slow`: advance both by 1. Return `p` — this is the cycle entry.

**Proof sketch:** let `a` = head→entry, `b` = entry→meet (forward in cycle), `L` = cycle length. At the meet, slow walked `a+b`, fast walked `2(a+b)`, and fast's lap surplus is a multiple of `L`, so `a+b ≡ 0 (mod L)` ⇒ `a ≡ L-b (mod L)`. `L-b` is exactly the forward distance from meet → entry. So walking `a` steps from head and `a` steps from meet (both at speed 1) lands both pointers on the entry.

## Code (clean canonical)
```cpp
ListNode* detectCycle(ListNode *head) {
    ListNode *slow = head, *fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) {                 // phase 1: cycle confirmed
            ListNode *p = head;             // phase 2: walk both at speed 1
            while (p != slow) {
                p = p->next;
                slow = slow->next;
            }
            return p;                       // cycle entry
        }
    }
    return nullptr;
}
```
**TC:** O(n)  **SC:** O(1)

## My Solution
```cpp
class Solution {
public:
    ListNode *detectCycle(ListNode *head) {
      ListNode *fast = head, *slow = head;
      while(fast && fast->next) {
        fast = fast->next->next;
        slow = slow->next;
        if(fast == slow) {
          fast = head;
          while(fast) {
            if(fast == slow) return fast;
            fast = fast->next;
            slow = slow->next;
          }
        }
      }
      return NULL;
    }
};
```

> **Refinements vs the canonical above:** my code reuses `fast` as the phase-2 walker — correct, but the name lies (it now moves at speed 1). Renaming to `p` makes the two phases self-documenting. Also flattened the inner `while(fast) { if(==) return ... }` to `while(p != slow) { ... }` — once a cycle is confirmed, the collision is guaranteed, so the extra null guard is dead defensive code. Behavior identical; reads more linearly.

## Other Approaches
- **Hash set of visited nodes** — walk from head, on first revisit return that node. O(n) / O(n). Mention as the intuitive baseline; interviewer will push for Floyd's.
- **Destructive marking** (mutate `val` or `next`) — forbidden by the "do not modify" constraint.

## Pitfalls
- **Phase 2 walks at speed 1, not 2** — easy to muscle-memory `fast = fast->next->next` again and quietly break it.
- Phase-2 reset point is **`head`**, not the meet — reset the *other* pointer, leave `slow` at the meet.
- `while (p != slow)` check is at the top: handles the `pos = 0` (entry is head) case correctly — both pointers already equal `head`, loop body doesn't execute, return `head` immediately.
- Empty list / single node with no cycle: outer guard `fast && fast->next` fails immediately → returns `nullptr`. No extra check needed.
- Compare after moving in phase 1, not before — otherwise the initial `slow == fast` (both `head`) returns spuriously.
- Don't confuse with [1. Linked List Cycle](1.%20Linked%20List%20Cycle.md): that one stops at phase 1 (true/false); this one *requires* phase 2 to locate the entry node.
