# 4. Palindrome Linked List (LC 234) — Easy
🔗 [Open on LeetCode](https://leetcode.com/problems/palindrome-linked-list/)

**Pattern:** Fast & Slow Pointers — find middle, then reverse-and-compare
**Trigger:** "is this singly linked list a **palindrome**" in **O(1) space** → can't index backwards, so find the middle (slow/fast) and reverse the second half in place

## Problem (1-liner)
Return `true` if a singly linked list reads the same forwards and backwards.
**Examples:** `[1,2,2,1]` → `true` · `[1,2]` → `false`

## Idea (1-liner)
A palindrome's second half, reversed, equals its first half. With a singly linked list you can't walk backwards, so use slow/fast to find the midpoint, reverse the back half in place, then compare the two halves node-by-node — and reverse it back to leave the list intact.

## Approach
1. Edge case: `0` or `1` node → `true`.
2. **Find midpoint:** `slow = fast = head`; while `fast->next && fast->next->next`, advance `slow` by 1 and `fast` by 2. `slow` lands at the end of the first half.
3. **Reverse second half:** `secondHalf = reverseList(slow->next)`.
4. **Compare:** walk `left = head` and `right = secondHalf` together; if any `val` differs, it's not a palindrome.
5. **Restore** (good citizenship): reverse the second half back before returning.

**Why `slow->next`:** reversing the node *after* the midpoint splits the list into two clean, non-overlapping halves (second half ≤ first half), so the middle node on odd length is never compared and the loop terminates on `right == nullptr`.

## Code (clean canonical)
```cpp
class Solution {
    ListNode* reverseList(ListNode* head) {
        ListNode* prev = nullptr;
        while (head) {
            ListNode* next = head->next;
            head->next = prev;
            prev = head;
            head = next;
        }
        return prev;
    }
public:
    bool isPalindrome(ListNode* head) {
        if (!head || !head->next) return true;

        // 1. find end of first half
        ListNode *slow = head, *fast = head;
        while (fast->next && fast->next->next) {
            slow = slow->next;
            fast = fast->next->next;
        }

        // 2. reverse second half
        ListNode* secondHalf = reverseList(slow->next);

        // 3. compare halves
        ListNode *left = head, *right = secondHalf;
        bool result = true;
        while (right) {
            if (left->val != right->val) { result = false; break; }
            left  = left->next;
            right = right->next;
        }

        // 4. restore list, then return
        reverseList(secondHalf);
        return result;
    }
};
```
**TC:** O(n)  **SC:** O(1)

## My Solution
```cpp
class Solution {
  ListNode* reverseList(ListNode* head) {
    ListNode* prev = NULL;
    while(head) {
      ListNode *next = head->next;
      head->next = prev;
      prev = head;
      head = next;
    }
    return prev;
  }
public:
    bool isPalindrome(ListNode* head) {
      if(!head || !head->next) return true;
      ListNode *fast = head, *slow = head;
      while(fast && fast->next) {
        fast = fast->next->next;
        slow = slow->next;
      }
      ListNode *newHead = reverseList(slow);
      ListNode *right = newHead;
      ListNode *left = head;
      while(right) {
        if(left->val != right->val) {
          reverseList(newHead);
          return false;
        } else {
          left = left->next;
          right = right->next;
        }
      }
      reverseList(newHead);
      return true;
    }
};
```

> **Correct, O(n)/O(1), and non-destructive** (restores the list — many submissions skip this). On odd length, `slow` lands on the true middle and that middle is harmlessly compared against itself. Refinements are stylistic only:
> - **Restore once, single return.** You call `reverseList(newHead)` in two places. `break` out with a `bool result` flag and restore once — drops the duplicated call and the `else`.
> - **Reverse `slow->next` instead of `slow`** (stop the walk one step earlier with `fast->next && fast->next->next`). Splits into two non-overlapping halves so the middle-node self-compare and the overlap subtlety disappear.
> - **`nullptr` over `NULL`** for consistency with modern C++.

## Other Approaches
- **Copy values to a `vector<int>`, two-pointer check** — O(n) / O(n). Simplest to write and doesn't touch the list, but fails the O(1)-space follow-up the interviewer will ask for.
- **Recursion** — use a forward pointer captured by reference while the call stack unwinds from the tail, comparing front-to-back. O(n) time but O(n) stack space, and easy to get wrong. Reverse-half is the expected answer.

## Pitfalls
- **Restore the list.** Reversing the second half mutates the input; reverse it back before returning or you've corrupted the caller's list (and any later test cases).
- **Loop guard for the midpoint:** `fast->next && fast->next->next` (this template) leaves `slow` at the **end of the first half** so `slow->next` starts the back half cleanly. Compare with [3. Middle of the Linked List](3.%20Middle%20of%20the%20Linked%20List.md) which uses `fast && fast->next` to land on the **second** middle — different goal, different guard.
- **Comparison loop drives on the shorter half** (`while (right)`): the reversed second half is ≤ the first, so iterating on `right` can't walk off the end of `left`.
- **Edge cases:** empty and single-node lists are palindromes — the early return handles both and avoids the midpoint logic.
- Don't compare `slow == fast` — this isn't cycle detection. Borrowed muscle memory from [1. Linked List Cycle](1.%20Linked%20List%20Cycle.md) is a common slip.
