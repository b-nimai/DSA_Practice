# 6. Remove Nth Node From End of List (LC 19) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/remove-nth-node-from-end-of-list/)

**Pattern:** Fast & Slow Pointers — fixed-gap two pointers (k-from-end variant)
**Trigger:** "remove the n-th node **from the end**" in **one pass** → pre-advance `fast` by `n`, then walk both until `fast` hits the end

## Problem (1-liner)
Given the head of a singly linked list, remove the n-th node counted from the end and return the head.
**Examples:** `[1,2,3,4,5]`, n=2 → `[1,2,3,5]` · `[1]`, n=1 → `[]` · `[1,2]`, n=1 → `[1]`

## Idea (1-liner)
Open a gap of `n` between `fast` and `slow`; when `fast` walks off the end, `slow` sits exactly on the node **before** the target — so to delete you need `slow` to land on the *predecessor*, which a dummy node guarantees even when the target is the head.

## Approach
1. `dummy → head`; `slow = fast = &dummy`.
2. Advance `fast` by `n + 1` steps (gap of n+1 so `slow` stops on the predecessor).
3. While `fast`: move both one step.
4. `slow->next = slow->next->next`; return `dummy.next`.

**Why the gap is `n+1`, not `n`:** to *delete* a node you must hold its predecessor. Starting both at a dummy and pre-advancing `fast` by `n+1` makes `slow` stop one node earlier than the target. Returning `dummy.next` then covers head-removal with **zero special cases**.

## Code (clean canonical)
```cpp
ListNode* removeNthFromEnd(ListNode* head, int n) {
    ListNode dummy(0, head);
    ListNode *slow = &dummy, *fast = &dummy;
    for (int i = 0; i <= n; i++) fast = fast->next;   // gap of n+1
    while (fast) { slow = slow->next; fast = fast->next; }
    slow->next = slow->next->next;
    return dummy.next;
}
```
**TC:** O(n) (single pass)  **SC:** O(1)

## My Solution
```cpp
class Solution {
public:
    ListNode* removeNthFromEnd(ListNode* head, int n) {
      ListNode *fast = head, *slow = head;
      // Forward the fast by Nth node
      for(int i = 0; i < n; i++) fast = fast->next;

      // If fast fell off the end, the node to remove is the head itself
      if(!fast) return head->next;

      while(fast->next) {
        fast = fast->next;
        slow = slow->next;
      }

      ListNode *temp = slow->next;
      slow->next = slow->next->next;
      delete(temp);
      
      return head;
    }
};
```

> **Bug fixed:** the original failed on `[1,2]`, n=2. After pre-advancing `fast` by `n`, when `n == length` `fast` becomes `nullptr` — the signal that the node to delete *is* the head. The old `if(slow == head && !head->next)` guard only caught the single-node list, not the general head-removal case. Adding `if(!fast) return head->next;` handles it; since `fast` is then guaranteed non-null, the loop guard simplifies from `fast && fast->next` to `fast->next`.
>
> **Refinement:** the dummy-node version above removes the `if(!fast)` branch entirely — the predecessor-via-dummy trick makes head-removal fall out of the same code path. Prefer it in interviews to avoid edge-case bugs.

## Other Approaches
- **Two-pass: count length, then walk to `len - n`** — O(n) / O(1) but two passes. Conceptually simplest; the one-pass two-pointer is the expected answer.
- **Push nodes to a vector, index `size - n`** — O(n) / O(n). Wasteful of space.

## Pitfalls
- **Gap off-by-one:** to delete you need the *predecessor*. With a dummy, advance `fast` by `n+1`; without a dummy (starting at `head`), advance by `n` and then special-case `fast == nullptr` ⇒ remove head.
- **Head removal:** the classic failure (e.g. `[1,2]`, n=2 or `[1]`, n=1). A dummy node makes it disappear; otherwise check `if(!fast) return head->next;` right after pre-advancing.
- **Pre-advance bounds:** constraints guarantee `1 ≤ n ≤ sz`, so `fast` never overruns during the `for`. Don't add a null check *inside* the pre-advance loop unless `n` could exceed length.
- **Return `dummy.next`, not `head`** — when the head itself is removed, the old `head` is dangling; `dummy.next` is the true new head.
