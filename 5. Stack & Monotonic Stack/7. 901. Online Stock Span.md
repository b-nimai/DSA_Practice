# 7. Online Stock Span (LC 901) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/online-stock-span/)

**Pattern:** Monotonic Stack — **previous-greater element, computed online (streaming) with span accumulation**
**Trigger:** "prices arrive **one at a time**; for each, how many **consecutive days back** (incl. today) were `≤` today?" = distance to the **previous strictly-greater** price → monotonic stack, popped while `top ≤ price`.

## Problem (1-liner)
Build a class `StockSpanner`; each `next(price)` feeds today's price and must **immediately** return its *span* = count of consecutive most-recent days (including today) whose price was `≤ today`.

**Example:** `100,80,60,70,60,75,85` → `1,1,1,2,1,4,6`.
`next(75)=4`: going back `60,70,60` are all `≤75`, then `80` stops it → today + 3 = 4.

## Idea (1-liner)
Span asks: *how far back until a price **strictly greater** than today?* That's the **previous-greater-element** distance. Re-scanning history each call is O(n²); instead keep a **decreasing** monotonic stack. Every price `≤` today can never be a future answer's blocker through today, so pop them — and **absorb their spans** so the count survives in O(1) amortized. "Online" = you can't see future prices, so all the work happens in `next`.

## Approach (canonical — store {price, span})
1. Stack holds `{price, span}` pairs, prices **strictly decreasing** bottom→top.
2. On `next(price)`: start `span = 1` (today itself).
3. While stack non-empty **and** `top.price ≤ price`: this older block is fully covered by today → `span += top.span`, pop it.
4. Push `{price, span}`; return `span`.
5. Each price is pushed once and popped at most once ⇒ **O(1) amortized** per call.

## Code (clean canonical)
```cpp
class StockSpanner {
    stack<pair<int,int>> st;          // {price, span}, prices strictly decreasing top-down
public:
    StockSpanner() {}

    int next(int price) {
        int span = 1;                              // today counts as 1
        while (!st.empty() && st.top().first <= price) {
            span += st.top().second;               // absorb spans of all days ≤ today
            st.pop();
        }
        st.push({price, span});
        return span;
    }
};
```
**TC:** O(1) amortized per `next` (O(n) total over n calls).  **SC:** O(n) — stack in the worst case (strictly decreasing prices).

## My Solution
**Index-based variant — correct.** ✓ Instead of accumulating spans, store `{price, dayIndex}` with a 1-based day counter `i`. After popping all `top.price ≤ price`, the new top is the **nearest previous strictly-greater day**, so the span is the index gap `i - top.second`; if the stack is empty, no greater day ever existed → span = `i` (all days so far).
```cpp
class StockSpanner {
    stack<pair<int,int>> st;          // {price, dayIndex}
    int i;
public:
    StockSpanner() { i = 0; }

    int next(int price) {
        while (!st.empty() && st.top().first <= price) st.pop();
        i++;                                                   // today's 1-based index
        int res = st.empty() ? i : i - st.top().second;       // distance to previous greater
        st.push({price, i});
        return res;
    }
};
```
**Span-accumulation vs index-gap:** both O(1) amortized, same stack size. Accumulation is more self-contained (no counter, no empty/non-empty branch) and is the editorial default — memorize it. The index version is the literal "previous greater element" framing and shows the link to 739/496 clearly.

## Other Approaches
- **Brute force per call** — on each `next`, walk backward over stored prices counting `≤ today` until a greater one. O(n) per call → O(n²) total; fine for n ≤ 10⁴ but the anti-pattern this problem teaches you to beat.
- **Two parallel arrays** — keep `prices[]` and a `span[]`; on each day jump backward via `j -= span[j]` (skip whole resolved blocks) instead of a stack. O(1) amortized, same idea as accumulation without the stack object — slicker but easier to off-by-one.

## Pitfalls
- **`≤` not `<` when popping** — equal prices *do* count toward the span ("less than **or equal**"). Popping on `<` only would undercount runs of equal prices.
- **Span counts today** — start at `1`, not `0`; forgetting this returns one less than the answer.
- **It's *previous* greater, not next** — direction flips vs 739. You absorb older days into today, you don't wait for a future day.
- **"Online" means no peeking ahead** — all logic lives in `next`; you can't precompute over the full array like 496/503.
- **Don't lose absorbed spans** — when popping a `≤` block you must add its span to today's, or you forget the days it represented (the whole point of storing the span/index).
- **`top()`/`pop()` on empty is UB** — guard `!st.empty()` first.
