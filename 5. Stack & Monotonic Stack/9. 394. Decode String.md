# 9. Decode String (LC 394) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/decode-string/)

**Pattern:** Stack — **nested `k[...]` decoding: save context on `[`, fold the repeated chunk back into its parent on `]`**
**Trigger:** "expand `k[encoded]`, possibly **nested** (`3[a2[c]]`)" → an inner block must finish before its outer repeat can apply = innermost-first = **LIFO** → stack the `(count, string-so-far)` context at each `[`, pop and fold on `]`.

## Problem (1-liner)
Decode a string where `k[encoded]` means *repeat `encoded` k times*; brackets can nest, `k` can be multi-digit, and letters can sit outside brackets.

**Examples:** `3[a]2[bc]` → `aaabcbc` · `3[a2[c]]` → `accaccacc` · `2[abc]3[cd]ef` → `abcabccdcdcdef`.

## Idea (1-liner)
Scan left→right building a `current` string. A digit extends a running `num`. On `[`, the current context (`num`, `current`) is **suspended** — push both, reset. On `]`, the just-finished `current` is the inner block: pop its repeat count `k` and its parent string, set `current = parent + current*k`. Nesting falls out for free because each `[` stacks one frame and each `]` unwinds exactly one. O(N_out).

## Approach (canonical — two stacks + running state)
1. State: `stack<int> counts`, `stack<string> strs`, plus `string current` and `int num`.
2. **digit** → `num = num*10 + (ch-'0')` (handles multi-digit `k`).
3. **`[`** → `counts.push(num); strs.push(current);` then reset `num=0, current=""` (start the inner block fresh).
4. **`]`** → pop `k=counts.top()`, pop `prev=strs.top()`; `current = prev + current repeated k times`.
5. **letter** → `current += ch`.
6. After the scan, `current` is the answer (no leftover stack).

## Code (clean canonical)
```cpp
class Solution {
public:
    string decodeString(string s) {
        stack<int> counts;
        stack<string> strs;
        string current = "";
        int num = 0;

        for (char ch : s) {
            if (isdigit(ch)) {
                num = num * 10 + (ch - '0');         // build multi-digit count in place
            } else if (ch == '[') {
                counts.push(num);                    // suspend parent context
                strs.push(current);
                num = 0; current = "";
            } else if (ch == ']') {
                int k = counts.top(); counts.pop();
                string prev = strs.top(); strs.pop();
                string repeated;
                repeated.reserve(current.size() * k);
                while (k--) repeated += current;     // append, never prepend
                current = prev + repeated;           // fold inner block into parent
            } else {
                current += ch;                       // ordinary letter
            }
        }
        return current;
    }
};
```
**TC:** O(N_out) — each output char built once (`reserve` avoids reallocs).  **SC:** O(N_out) worst case + O(depth) stacks.

## My Solution
**Single stack of strings — correct after three fixes.** ✓ Pushes every char as its own 1-char string; on `]` pops back to `[` to rebuild the inner block, pops the `[`, then reads digit-strings and re-`stoi`s the count. The three bugs hit while writing it (all structurally impossible in the two-stack version): `to_string(ch)` pushed the char's *code* not the char (`string(1,ch)` fixes); `isdigit(st.top())` needs `st.top()[0]` (top is a `string`); and **forgetting `st.pop()` for the `[`** left it on top so the digit loop ran zero times → `stoi("")` threw `std::invalid_argument`.
```cpp
class Solution {
public:
    string decodeString(string s) {
        stack<string> st;
        for (char ch : s) {
            if (ch != ']') st.push(string(1, ch));
            else {
                string substr = "";
                while (st.top() != "[") { substr = st.top() + substr; st.pop(); }
                st.pop();                                    // ← drop the "[" (missing = stoi("") crash)
                string k = "";
                while (!st.empty() && isdigit(st.top()[0])) { k = st.top() + k; st.pop(); }
                for (int i = 0; i < stoi(k); i++) st.push(substr);
            }
        }
        string result = "";
        while (!st.empty()) { result = st.top() + result; st.pop(); }
        return result;
    }
};
```
**Two-stack vs single-stack:** both O(N_out) time, but the single-stack version pushes every char and rebuilds via **prepend** (`a + b`) → O(n²)-ish copying, parses the number twice, and routes numbers/`[`/letters through one `string` type (the source of all three bugs). The two-stack version separates count from text, builds `num` arithmetically (no `stoi`), and **appends** with `reserve`. Prefer two-stack as the canonical.

## Other Approaches
- **Recursion / recursive-descent parser** — carry an index by reference; on `[` recurse to decode the inner block, on `]` return. Same complexity, arguably cleanest; the two-stack version is just this with an explicit stack.
- **Single stack of strings (My Solution)** — works but prepend-concatenation is O(n²) and the count is parsed twice; fine for `s.length ≤ 30` but not the template to memorize.
- **Two stacks (canonical)** — the iterative sweet spot: no recursion depth risk, no `stoi`, append-only.

## Pitfalls
- **Multi-digit `k`** — `num = num*10 + (ch-'0')`, not a single `ch-'0'`; `12[a]` must read `12`, not `2`. (In the string version, read **all** consecutive digit chars.)
- **Reset `num` AND `current` on `[`** — forgetting either bleeds the parent's count/text into the child block.
- **`]` folds into the parent** — `current = prev + current*k`, not just `current*k`; dropping `prev` loses everything before the bracket (e.g. the `a` in `a3[b]`).
- **Single-stack: pop the `[`** — after rebuilding the inner block you must `pop()` the `[`, else the digit reader sees `[`, count stays empty, and `stoi("")` **throws** at runtime.
- **Build the count arithmetically, not via `stoi` on stacked chars** — avoids the empty-string `stoi` crash and the double parse.
- **Append, don't prepend** — `res = top + res` in a loop is O(n²); append + `reserve`, or build forward and reverse once.
- **`isdigit` on a `string`** — `isdigit` takes a `char`; pass `st.top()[0]`, or better, test the raw `char ch` before it ever becomes a string.
