## When to reach for Stack / Monotonic Stack
1. **Matching / nesting** вАФ parentheses, tags, "push state on open, pop on close" parsers (20, RPN, simplify path, decode string).
2. **Next / previous greater-or-smaller element** вАФ monotonic stack of indices; pop while the invariant breaks.
3. **Histogram / rectangles / stock span / temperatures** вАФ span between an element and its next-greater/smaller boundary.

## Common Pitfalls Across the Pattern
- **`top()` / `pop()` on an empty stack is UB** вАФ always guard `!st.empty()` before touching the top.
- **Match the type, not just presence** вАФ a closer must cancel the *right* opener; "stack non-empty" alone lets `"([)]"` slip through.
- **Don't forget the post-loop check** вАФ leftover items (unclosed openers, unresolved indices) are only visible after the scan; returning `true` unconditionally is the classic bug.
- **Store indices, not values, in a monotonic stack** when you need spans/distances (so you can compute `i - st.top()`).
