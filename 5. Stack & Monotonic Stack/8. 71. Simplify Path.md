# 8. Simplify Path (LC 71) ‚Äî Medium
üîó [Open on LeetCode](https://leetcode.com/problems/simplify-path/)

**Pattern:** Stack ‚Äî **tokenize on `/`, push directories, `..` pops the last one**
**Trigger:** "canonicalize a Unix path ‚Äî collapse `//`, drop `.`, make `..` go up a level" ‚Üí split on `/`; a real name **pushes**, `..` **pops** the most recent dir ‚Üí that LIFO "undo the last step" is a **stack**.

## Problem (1-liner)
Given an absolute Unix path, return its canonical form: single slashes, no trailing slash (except root), and `.`/`..` resolved.

**Examples:** `/home/` ‚Üí `/home` ¬∑ `/home//foo/` ‚Üí `/home/foo` ¬∑ `/a/./b/../c` ‚Üí `/a/c` ¬∑ `/../` ‚Üí `/` ¬∑ `/.../a/../b/c/../d/./` ‚Üí `/.../b/d` (`...` is a *valid name*).

## Idea (1-liner)
Walk the path splitting into segments at every `/`. Each segment is one of four things: empty (`//` or trailing `/`) ‚Üí ignore; `.` (current dir) ‚Üí ignore; `..` (parent) ‚Üí **pop** the last pushed dir if any; anything else (incl. `...`, `....`, `_`, names) ‚Üí **push**. The stack naturally holds the surviving directories top-to-bottom; join with `/` at the end. O(n).

## Approach (canonical)
1. Append a **sentinel `/`** to `path` so the final segment flushes inside the loop (kills the "last word after the loop" special case).
2. Build `word` char by char; on each `/`, classify the completed `word`:
   - empty ‚Üí skip ¬∑ `.` ‚Üí skip ¬∑ `..` ‚Üí `pop_back()` **iff** non-empty ¬∑ else ‚Üí `push_back(word)`.
3. Reset `word` and continue.
4. Join the stack front‚Üíback with `/`; if empty, return `"/"` (root).

## Code (clean canonical)
```cpp
class Solution {
public:
    string simplifyPath(string path) {
        path += '/';                                   // sentinel flushes the last segment
        vector<string> st;                             // surviving directories (used as a stack)
        string word;
        for (char ch : path) {
            if (ch != '/') { word += ch; continue; }   // still building a segment
            if (!word.empty()) {
                if (word == "..") { if (!st.empty()) st.pop_back(); }  // up one level (clamp at root)
                else if (word != ".") st.push_back(word);              // real name; "." ignored
            }
            word.clear();
        }
        string result;
        for (string& dir : st) { result += '/'; result += dir; }
        return result.empty() ? "/" : result;          // empty stack ‚áí root
    }
};
```
**TC:** O(n) ‚Äî each char visited once, each segment pushed/popped at most once.  **SC:** O(n) ‚Äî stack + output.

## My Solution
**Same stack idea, with a `deque` ‚Äî correct after two fixes.** ‚úì Originally bundled the pop into one condition `word == ".." && !q.empty()` (else-pushed `..` when the stack was empty ‚Üí `/../` gave `/..`) and never handled `.`; also the trailing-word block re-introduced the bug for paths not ending in `/` (`‚Ä¶/..` ‚Üí kept `..`). Fixed by (a) splitting the three cases explicitly and (b) appending a sentinel `/` so the last segment flushes in-loop instead of in a duplicated block.
```cpp
class Solution {
    deque<string> q;                                   // ‚öÝ should be LOCAL, not a member (state leaks across calls)
public:
    string simplifyPath(string path) {
        path += "/";
        string word = "";
        for (char ch : path) {
            if (ch == '/') {
                if (!word.empty()) {
                    if (word == ".");                  // ‚öÝ dangling-semicolon foot-gun
                    else if (word == "..") { if (!q.empty()) q.pop_back(); }
                    else q.push_back(word);
                }
                word = "";
            } else { word += ch; }
        }
        string result = "";
        while (!q.empty()) { result += '/'; result += q.front(); q.pop_front(); }
        return result.empty() ? "/" : result;
    }
};
```
**deque vs vector:** only `push_back`/`pop_back` + ordered read are used ‚Üí a `vector<string>` is the natural stack (contiguous, lighter) and reads front‚Üíback directly; `deque` here is overkill. Also the scratch container should be a **local**, not a class member (a member persists between calls and would corrupt a second test).

## Family map
- **Stack's matching/parsing half** ‚Üí [[1-20-valid-parentheses]] / [[3-150-evaluate-reverse-polish-notation]] ‚Äî same "push state, resolve on a delimiter" shape; here the delimiter is `/` and `..` is the resolver.

- **Not monotonic** ‚Üí unlike [[4-739-daily-temperatures]] / [[7-901-online-stock-span]], no ordering invariant; it's a plain LIFO, popping is driven by the `..` token, not a comparison.

## Other Approaches
- **`stringstream` + `getline(ss, tok, '/')`** ‚Äî same algorithm, splitting delegated to the stream. Cleaner parse, identical O(n)/O(n); just a style choice over the manual `word` accumulator.
- **In-place, O(1) extra space** ‚Äî overwrite the input with a write-pointer instead of a separate container. Saves the auxiliary stack but is fiddly (must walk back on `..`); not worth it for `n ‚â§ 3000`.
- **`split` + functional filter/reduce** (other langs) ‚Äî split on `/`, fold over tokens. Same complexity; C++ has no built-in split so the manual loop wins for clarity.

## Pitfalls
- **`..` on an empty stack must do nothing** ‚Äî you can't go above root (`/../` ‚Üí `/`). Don't fold the emptiness check into the same condition that decides push-vs-pop, or an empty-stack `..` silently gets pushed.
- **The last segment after the final `/`** ‚Äî if the path doesn't end in `/` (`/a/b/..`), the last `word` never hits a `/`. Either flush it in a post-loop block **with the same logic**, or (cleaner) append a sentinel `/`. Duplicating the logic is how the `..`/`. ` bugs crept back.
- **`.` must be ignored, not pushed** ‚Äî `else if (word != ".")` before `push_back`, or `.` ends up in the output.
- **`...`, `....`, `_` are valid names** ‚Äî only exactly `.` and `..` are special; don't over-match periods.
- **Keep scratch state local** ‚Äî a class-member `deque`/`vector` persists across calls and corrupts later inputs.
- **Avoid the empty-statement `if (cond);`** ‚Äî the dangling `;` makes the next line look guarded when it isn't; prefer an explicit skip or fold into `else if`.
- **Empty result ‚áí return `"/"`** ‚Äî root (and paths that pop everything) leave the stack empty.
