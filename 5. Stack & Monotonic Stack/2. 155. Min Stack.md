# 2. Min Stack (LC 155) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/min-stack/)

**Pattern:** Stack — **augmented stack / carry an invariant alongside each element**
**Trigger:** "stack that also returns its **minimum in O(1)**" → a plain stack can't, because popping the min needs the *next* min instantly → **store the running min next to each value** (pair-stack), or encode it into the value itself (O(1)-space trick)

## Problem (1-liner)
Design `MinStack` supporting `push`, `pop`, `top`, and `getMin` — **all O(1)**. `pop/top/getMin` are only called on a non-empty stack.
**Example:** `push(-2), push(0), push(-3)` → `getMin()=-3` → `pop()` → `top()=0`, `getMin()=-2`.

## Idea (1-liner)
`getMin` must be O(1), but the min can change on every `push`/`pop`, so you can't recompute it (that's O(n)). The fix is to **remember, for every element, what the minimum was at the moment it sat on top**. Then the current min is always readable from the top in O(1), and popping automatically exposes the previous min. The pair-stack stores that explicitly; the encoding trick stores it implicitly to save space.

## Approach (canonical — pair stack)
1. Stack of `pair<int,int>` = `{value, minAtOrBelowThis}`.
2. **push(v):** the new running min is `min(v, currentMin)`; if empty it's just `v`. Push `{v, newMin}`.
3. **pop:** plain `st.pop()` — the pair below already carries the correct min, so it's exposed for free.
4. **top:** `st.top().first`. **getMin:** `st.top().second`.
5. Every op is O(1); space is O(n) (one extra int per element).

## Code (clean canonical)
```cpp
class MinStack {
    stack<pair<int,int>> st;                 // {value, min of everything at/below}
public:
    MinStack() {}

    void push(int v) {
        int mn = st.empty() ? v : min(v, st.top().second);
        st.push({v, mn});
    }
    void pop()      { st.pop(); }             // pair below carries the prior min
    int top()       { return st.top().first;  }
    int getMin()    { return st.top().second; }
};
```
**TC:** O(1) per operation.  **SC:** O(n) — one extra `int` per element.

## My Solution
**(a) Pair stack — correct, matches the canonical.** ✓ The clean, safe answer.
```cpp
class MinStack {
    stack<pair<int, int>> st;
public:
    MinStack() {}
    void push(int value) {
        if (st.empty()) st.push({value, value});
        else st.push({value, min(value, st.top().second)});
    }
    void pop()   { if (!st.empty()) st.pop(); }
    int top()    { if (st.empty()) return -1; return st.top().first; }
    int getMin() { if (!st.empty()) return st.top().second; return -1; }
};
```

**(b) O(1)-space encoding trick — clever; had an overflow bug, now fixed.** Pushes an encoded sentinel `2*value - mini` when a new min arrives; on pop, a value below `mini` decodes the previous min via `2*mini - x`.

⚠️ **Original bug:** `int x = st.top();` **truncated** the 64-bit encoded sentinel to 32 bits, so the restore guard `x < mini` silently failed on extreme inputs — `push(INT_MAX); push(INT_MIN); pop();` then `getMin()` returned `INT_MIN` instead of `INT_MAX`. **Fixed below:** `stack<long long>`, `long long x`, `1LL`.
```cpp
class MinStack {
    stack<long long> st;                      // long long (long is 32-bit on MSVC)
    long long mini;
public:
    MinStack() { mini = LONG_MIN; }
    void push(int value) {
        if (st.empty()) { st.push(value); mini = value; }
        else if (value < mini) {
            st.push(1LL * 2 * value - mini);  // sentinel < value flags "new min here"
            mini = value;
        } else st.push(value);
    }
    void pop() {
        if (st.empty()) return;
        long long x = st.top();               // FIX: was `int x` — the bug
        st.pop();
        if (x < mini) mini = 2 * mini - x;    // sentinel ⇒ decode the previous min
    }
    int top() {
        if (st.empty()) return -1;
        if (mini > st.top()) return mini;     // top is a sentinel ⇒ real value is `mini`
        return st.top();
    }
    int getMin() { return mini; }
};
```
**Why the sentinel can exceed `int`:** with `value = INT_MIN`, `mini = INT_MAX`, `2*value - mini ≈ -6.4e9` — well outside `int`, hence the mandatory 64-bit width.

## Family map
- **Augmented-stack** (this) → carry *any* O(1)-queryable aggregate beside each element; same shape as a **max stack**, or a stack tracking a running GCD/sum.
- **Monotonic stack** (the pattern's other half) → [[4-daily-temperatures]], next-greater family — there the *min/max is the stack's structure*, not a piggybacked field.
- **Two-stack composition** → a "queue with O(1) min" lifts this idea across two stacks.

## Other Approaches
- **Two stacks** — a main stack + a `minStack` that pushes the new min on each `push` and pops in lockstep. Same O(n) space, arguably the most readable. Optimization: push to `minStack` only when `v <= currentMin` and pop only when `v == minStack.top()` (handles duplicate mins).
- **Single stack of just values + separate `min` var** — fails: when you pop the current min you can't recover the previous one. That gap is exactly what the pair / sentinel / second-stack repairs.
- **Recompute min on getMin** — O(n) per `getMin`, violates the O(1) requirement.

## Pitfalls
- **Encoding trick: 64-bit everywhere.** The sentinel `2*value - mini` overflows `int`; the stack, the popped temp `x`, and the literal (`1LL`/`2LL`) must all be 64-bit. The original `int x = st.top()` truncation was the live bug.
- **`long` is not 64-bit on Windows/MSVC** (LeetCode's judge). Use `long long`, not `long`, or the sentinel overflows there too.
- **Single-min-variable trap** — tracking only the current min without per-element history can't restore the min after popping it. You *must* store history (pair / second stack / sentinel).
- **Duplicate minimums** in the two-stack variant — push to `minStack` on `<=` (not `<`) and pop on `==`, or popping one copy of a repeated min corrupts the tracked minimum.
- **Pattern seed:** "stack that answers an O(1) aggregate query" generalizes — swap `min` for `max`, or compose two of these for an O(1)-min **queue**. The invariant *"store the aggregate as-of-this-element"* is the transferable idea.
