# 6. 3Sum (LC 15) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/3sum/)

**Pattern:** Two Pointers — sort + outer loop + inner two-pointer, skip duplicates
**Trigger:** find triplets / k-sum with no duplicates; reduce k-sum to (k-1)-sum

## Problem (1-liner)
Return all unique triplets `[a, b, c]` with `a + b + c == 0`.
**Example:** `[-1,0,1,2,-1,-4]` → `[[-1,-1,2], [-1,0,1]]`

## Idea (1-liner)
Sort. Fix `i`; on the rest, use two-pointer for `target = -nums[i]`. Skip dups at every level.

## Approach
1. Sort `nums`.
2. For `i = 0..n-3`:
   - Skip if `i > 0 && nums[i] == nums[i-1]`.
   - `l = i+1`, `r = n-1`, `target = -nums[i]`.
   - Loop `l < r`:
     - `sum == target` → push `{nums[i], nums[l], nums[r]}`; `l++; r--;` then skip dups on both sides.
     - `sum < target` → `l++`.
     - else → `r--`.

## Code (clean canonical)
```cpp
vector<vector<int>> threeSum(vector<int>& nums) {
    sort(nums.begin(), nums.end());
    vector<vector<int>> res;
    int n = nums.size();
    for (int i = 0; i < n - 2; i++) {
        if (i > 0 && nums[i] == nums[i - 1]) continue;
        int l = i + 1, r = n - 1, t = -nums[i];
        while (l < r) {
            int s = nums[l] + nums[r];
            if (s == t) {
                res.push_back({nums[i], nums[l], nums[r]});
                l++; r--;
                while (l < r && nums[l] == nums[l - 1]) l++;
                while (l < r && nums[r] == nums[r + 1]) r--;
            } else if (s < t) l++;
            else r--;
        }
    }
    return res;
}
```
**TC:** O(n²)  **SC:** O(1) extra (output not counted; sort uses O(log n) stack)

## My Solution
```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        vector<vector<int>>res;
        sort(nums.begin(), nums.end());
        int size = nums.size();
        for(int i = 0; i < size - 2; i++) {
          if(i > 0 && nums[i] == nums[i - 1]) continue;
          int start = i + 1, end = size - 1;
          while(start < end) {
            int sum = nums[i] + nums[start] + nums[end];
            if(sum == 0) {
              res.push_back({nums[i], nums[start], nums[end]});
              end--;
              start++;
              while(start < end && nums[end] == nums[end - 1]) end--;
              while(start < end && nums[start] == nums[start - 1]) start++;
            }
            else if(sum > 0) end--;
            else start++;
          } 
        }
        return res;
    }
};
```

## Other Approaches
- **Brute force:** all triplets + dedup via set → O(n³) / O(n)
- **HashSet for inner pair:** sort + outer + hashset → O(n²) / O(n)
- **Generic k-Sum recursion:** O(n^(k-1))

## Pitfalls
- **Must sort first** — both for two-pointer and dup skipping
- Dedup at **3 places**: outer `i`, inner `l` after match, inner `r` after match
- Early break: if `nums[i] > 0` → no triplet sums to 0 (optional pruning)
- Use `long long` if values may be huge
