# 4. Move Zeroes (LC 283) — Easy
🔗 [Open on LeetCode](https://leetcode.com/problems/move-zeroes/)

**Pattern:** Two Pointers — slow/fast partition
**Trigger:** in-place partition / move target value to end while preserving order

## Problem (1-liner)
Move all `0`s to the end **in-place**; preserve order of non-zeros.
**Example:** `[0,1,0,3,12]` → `[1,3,12,0,0]`

## Idea (1-liner)
`write` tracks next non-zero slot. Swap `nums[read]` into `nums[write]` whenever `nums[read] != 0`.

## Approach
1. `write = 0`.
2. For `read = 0..n-1`:
   - If `nums[read] != 0` → `swap(nums[read], nums[write++])`.

## Code (clean canonical)
```cpp
void moveZeroes(vector<int>& nums) {
    int write = 0;
    for (int read = 0; read < nums.size(); read++) {
        if (nums[read] != 0)
            swap(nums[read], nums[write++]);
    }
}
```
**TC:** O(n)  **SC:** O(1)

## My Solution
```cpp
class Solution {
public:
    void moveZeroes(vector<int>& nums) {
      int index = 0;
      for(int i = 0; i < nums.size(); i++) {
        if(nums[i] != 0) {
          swap(nums[i], nums[index]);
          index++;
        }
      }
    }
};
```

## Other Approaches
- **Two-pass overwrite:** copy non-zeros to front, then fill rest with `0` → O(n) / O(1) (more writes)
- **Extra array:** build new array → O(n) / O(n) (violates in-place)

## Pitfalls
- Swap, don't just copy — keeps the array consistent in one pass
- When `read == write`, swap is a no-op (still correct)
- Order of non-zeros must be preserved (so don't sort)
