# 5. Container With Most Water (LC 11) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/container-with-most-water/)

**Pattern:** Two Pointers — opposite ends, shrink shorter wall
**Trigger:** maximize area / pair selection where one side is the bottleneck

## Problem (1-liner)
Pick two lines so the rectangle they form with x-axis holds max water; return that area.
**Example:** `[1,8,6,2,5,4,8,3,7]` → `49`

## Idea (1-liner)
`l, r` at ends. Area = `min(h[l], h[r]) * (r - l)`. Move the **shorter** wall inward — the longer one can never give more water at smaller width.

## Approach
1. `l = 0`, `r = n - 1`, `best = 0`.
2. Loop while `l < r`:
   - `area = min(h[l], h[r]) * (r - l)`; update `best`.
   - If `h[l] < h[r]` → `l++`; else → `r--`.

## Code (clean canonical)
```cpp
int maxArea(vector<int>& h) {
    int l = 0, r = h.size() - 1, best = 0;
    while (l < r) {
        int area = min(h[l], h[r]) * (r - l);
        best = max(best, area);
        (h[l] < h[r]) ? l++ : r--;
    }
    return best;
}
```
**TC:** O(n)  **SC:** O(1)

## My Solution
```cpp
class Solution {
public:
    int maxArea(vector<int>& height) {
        int maxi = 0;
        int start = 0, end = height.size() - 1;
        while (start < end) {
            int water = min(height[start], height[end]) * (end - start);
            maxi = max(maxi, water);
            if (height[start] < height[end]) start++;
            else end--;
        }
        return maxi;
    }
};
```

## Other Approaches
- **Brute force:** all pairs `(i, j)` → O(n²) / O(1)

## Why move the shorter wall?
Width shrinks no matter who moves. Keeping the shorter wall caps area at `short * width`. Only by replacing the shorter wall can area possibly grow.

## Pitfalls
- Move the **shorter** side, not arbitrary
- If `h[l] == h[r]`, moving either is fine (problem-equivalent)
- This problem ≠ Trapping Rain Water (no slanting; pick exactly 2 lines)
