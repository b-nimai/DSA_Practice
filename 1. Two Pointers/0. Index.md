## Variants of Two Pointers (cluster the problems)

| Variant | Problems | Signature |
|---------|----------|-----------|
| **Opposite ends** | 1, 2, 5, 9, 10 | `l = 0, r = n-1; while (l < r)` |
| **Slow/Fast (same direction)** | 3, 4 | `write` lags `read`; in-place rewrite |
| **Sort + outer loop(s) + inner 2-ptr** | 6, 11, 12 | reduce k-sum to (k-1)-sum |
| **3-way partition (Dutch flag)** | 7 | `lo/mid/hi` regions in one pass |
| **Range two-pointer (numeric)** | 8 | walk integer range, no array |

## When to reach for Two Pointers
1. Array/string is **sorted** (or you can sort).
2. You're searching for a **pair / triplet** satisfying a relation.
3. You need an **in-place** O(1) extra-space operation.
4. The problem has **symmetry** (palindrome, reverse-style).
5. The signal "shrink the shorter side" makes monotone progress.

## Common Pitfalls Across the Pattern
- Off-by-one: `l < r` vs `l <= r` (range 2-ptr like LC 633 needs `<=`).
- Forgetting to **skip duplicates** after a hit (3Sum: 3 places; 4Sum: after `i`, after `j` with `j > i+1`, and both pointers).
- In 3Sum Closest, move the pointers by **current `sum`** vs target — not by the best-so-far `closest`.
- **Integer overflow** in `sum` / `square`: cast operand to `long long` *before* multiply.
- 1-indexed vs 0-indexed return values.
- Don't `++mid` after swapping with `hi` in Dutch flag — value is unknown.
- In Trapping Rain Water, must always update the **chosen side's** max before adding to water.

## C++ Two Pointers template
```cpp
int l = 0, r = n - 1;
while (l < r) {
    // examine (a[l], a[r]) — usually with a sum/min/max
    if (/* shrink condition on left */) l++;
    else if (/* shrink condition on right */) r--;
    else { /* hit: record, then move both, skip dups if needed */ l++; r--; }
}
```
