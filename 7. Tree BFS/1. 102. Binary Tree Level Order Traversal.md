# 1. 102. Binary Tree Level Order Traversal (LC 102) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/binary-tree-level-order-traversal/)

**Pattern:** Tree BFS — the **founding skeleton** of the whole pattern. Queue + a **level-size snapshot** (`int sz = q.size()`) taken *before* draining the level, so each inner loop processes exactly one level and you can group values per level.
**Trigger:** "**level order**", "level by level", "group nodes by depth", "process the tree breadth-first" → queue, snapshot `q.size()`, drain that many, push their children, repeat.

## Problem (1-liner)
Return the node values **grouped by level**, top-to-bottom, left-to-right: `vector<vector<int>>`.
**Examples:** `[3,9,20,null,null,15,7]` → `[[3],[9,20],[15,7]]` · `[1]` → `[[1]]` · `[]` → `[]`.
**Constraints:** `0 … 2000` nodes, values `[-1000, 1000]`. Empty tree is legal → must return `[]`.

## Idea (1-liner)
A queue naturally yields nodes in breadth-first order, but that's one flat stream — to know **where one level ends**, snapshot `q.size()` *before* the inner loop: at that instant the queue holds **exactly** the current level, so looping that many times drains precisely one level while the children you push form the next.

## Approach
1. Guard the empty tree (`!root → []`), push `root`.
2. While the queue is non-empty:
   - `int sz = q.size();` — **freeze the level boundary** (do *not* read `q.size()` inside the loop; it grows as you push children).
   - Loop `sz` times: pop a node, record its value into a per-level `level` vector, push its **non-null** children.
   - Append `level` to the result.
3. Return the result.

> The single load-bearing line is `int sz = q.size();` **before** the inner loop. Everything else is plumbing. Get that snapshot wrong and levels bleed into each other.

## Code (clean canonical)
```cpp
class Solution {
public:
    vector<vector<int>> levelOrder(TreeNode* root) {
        vector<vector<int>> result;
        if (!root) return result;

        queue<TreeNode*> q;
        q.push(root);

        while (!q.empty()) {
            int sz = q.size();              // snapshot: queue == this level right now
            vector<int> level;
            level.reserve(sz);             // we know the exact level width

            for (int i = 0; i < sz; ++i) {
                TreeNode* node = q.front(); q.pop();
                level.push_back(node->val);
                if (node->left)  q.push(node->left);
                if (node->right) q.push(node->right);
            }
            result.push_back(move(level));
        }
        return result;
    }
};
```
**TC:** O(n) — each node enqueued/dequeued once.  **SC:** O(n) — queue holds up to the widest level (~n/2 leaves) + the O(n) output.

## My Solution (passed all tests — refinement notes)
My original was already the reference implementation — correct, O(n)/O(n), idiomatic. The version above only **polishes**:
- **`temp.reserve(sz)`** — the level width is known up front, so pre-sizing the per-level vector avoids its internal reallocations. Free win, negligible at n ≤ 2000 but the "right" habit.
- **`move(level)` into the result** — avoids copying the level vector on push. Pure micro-optimization.
- **`temp` → `level`** — the vector *is* one level; the name documents intent.

Everything correct in the original was kept: the empty-tree guard, pushing only non-null children, and — most importantly — capturing `int size = q.size();` **before** the inner loop. Nothing was wrong; these are taste/perf only.

## Other Approaches
- **DFS with a depth parameter** — recurse carrying `depth`; if `depth == result.size()` push a new empty level, then `result[depth].push_back(val)`, recurse left then right. Produces the same grouped output in O(n)/O(h) — handy when recursion is more natural, but BFS is the canonical match for "level order."
- **Two queues / level sentinel (`nullptr` marker)** — alternative ways to detect level boundaries; the `q.size()` snapshot is cleaner and the interview default. Avoid the sentinel version unless you have a reason.

## Pitfalls
- **Snapshot `q.size()` *before* the inner loop**, into a local. Reading `q.size()` in the loop condition is the classic bug — the queue grows as you push children, so the boundary keeps moving and levels merge.
- **Handle the empty tree** — `!root` must return `[]` (constraints allow 0 nodes). Skipping the guard and pushing a null root corrupts the first pop.
- **Push only non-null children** here — keeps the queue clean so `q.size()` equals the real level width. (Different from 297, where nulls are deliberately kept.)
- **Don't forget to append the level vector** after the inner loop — easy to build `level` and never `push_back` it.
- **Order is left-then-right** when pushing children — reversing them silently mirrors every level.
