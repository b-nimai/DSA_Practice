# 5. 637. Average of Levels in Binary Tree (LC 637) — Easy
🔗 [Open on LeetCode](https://leetcode.com/problems/average-of-levels-in-binary-tree/)

**Pattern:** Tree BFS — **102 skeleton, reduce each level to one aggregate**. Same level-size-snapshot loop; instead of collecting the whole level vector, you fold it into a single number (here, `sum / count`).
**Trigger:** "**average / sum / max / min of each level**", "**per-level aggregate**", "one number per depth" → plain level-order BFS, accumulate over the level, push the aggregate. (Right View = same shape but keeps `q.back()`; this keeps `sum/size`.)

## Problem (1-liner)
Return an array where index `i` = the **average of all node values at level `i`**; answers within `1e-5` are accepted.
**Examples:** `[3,9,20,null,null,15,7]` → `[3, 14.5, 11]` · single node `[5]` → `[5]`.
**Constraints:** `1 … 10^4` nodes (**tree is never empty**), values `-2^31 … 2^31-1`. Note the full-`int`-range values — see overflow trap below.

> **The load-bearing trick is the sum type.** A level can hold ~5000 nodes each near `2^31`, so an `int`/`long` running sum overflows. Accumulate into a `double` (or `long long`) so each `int` value is widened *before* it's added.

## Idea (1-liner)
Run plain left→right BFS. Snapshot `size = q.size()` per level, sum the `size` node values, push `sum / size`.

## Approach
1. Seed queue with `root` (guaranteed non-null by constraints).
2. Per level: snapshot `size = q.size()`, reset `sum = 0` as a **`double`**.
3. Drain `size` nodes: add each `node->val` to `sum`, enqueue non-null children left→right.
4. Push `sum / size` to the result.

## Code (clean canonical)
```cpp
class Solution {
public:
    vector<double> averageOfLevels(TreeNode* root) {
        vector<double> result;
        queue<TreeNode*> q;
        q.push(root);

        while (!q.empty()) {
            int size = q.size();
            double sum = 0;                 // double so int values widen before adding (no overflow)

            for (int i = 0; i < size; ++i) {
                TreeNode* node = q.front(); q.pop();
                sum += node->val;
                if (node->left)  q.push(node->left);
                if (node->right) q.push(node->right);
            }
            result.push_back(sum / size);   // double / int → double division
        }
        return result;
    }
};
```
**TC:** O(n) — each node visited once.  **SC:** O(n) — queue holds up to a full level (worst case ~n/2 leaves).

My version was correct as-is: `double sum` already dodges the overflow trap, `size` snapshotted before the inner loop, non-null children only. Only cosmetic spacing tightened (`vector<double> result;`, `queue<TreeNode*> q;`). No asymptotic or correctness change.

## Other Approaches
- **DFS, depth-indexed (O(h) space).** Recurse carrying `depth`; maintain `vector<double> sum` and `vector<int> cnt`, do `sum[depth] += val; cnt[depth]++`, then divide element-wise at the end. Same O(n) time; trades the O(width) queue for an O(height) recursion stack — better on tall/balanced trees. Reach for it when asked to cut BFS queue space.
  ```cpp
  void dfs(TreeNode* n, int d, vector<double>& sum, vector<int>& cnt) {
      if (!n) return;
      if (d == (int)sum.size()) { sum.push_back(0); cnt.push_back(0); }
      sum[d] += n->val; cnt[d]++;
      dfs(n->left, d + 1, sum, cnt);
      dfs(n->right, d + 1, sum, cnt);
  }
  ```
- **BFS with `long long` sum + manual cast.** `long long sum = 0; ... result.push_back((double)sum / size);`. Also overflow-safe; the `double`-sum form is just one fewer cast.

## Pitfalls
- **Integer overflow on the sum.** Values span the full `int` range and a level can be wide — sum into a **`double`** (or `long long`), never a bare `int`. This is the one real trap of an otherwise-trivial problem.
- **Division type.** `sum / size` only gives a real average because `sum` is `double`; with an integer sum you'd need `(double)sum / size`.
- Inherited from 102: **snapshot `q.size()` before the inner loop**, **push only non-null children**, **left-before-right** enqueue.
- No empty-tree guard needed here (constraints guarantee ≥1 node) — but add `if(!root) return {};` if reusing this template elsewhere.
