# 6. 111. Minimum Depth of Binary Tree (LC 111) — Easy
🔗 [Open on LeetCode](https://leetcode.com/problems/minimum-depth-of-binary-tree/)

**Pattern:** Tree BFS — **102 skeleton, but stop at the first leaf**. The shortest root→leaf path is found the instant level-order BFS dequeues a node with no children. (Also solvable with DFS, but BFS is the pattern-true answer because "minimum / shortest in an unweighted tree" = BFS short-circuit.)
**Trigger:** "**minimum depth**", "**shortest** path to the **nearest leaf**", "fewest nodes root→leaf" → level-order BFS, return `depth` at the **first leaf** popped. The one-child trap (a node with a single child is **not** a leaf) is what separates this from `maxDepth`.

## Problem (1-liner)
Return the number of nodes on the **shortest** path from root down to the **nearest leaf** (a leaf = node with **no** children).
**Examples:** `[3,9,20,null,null,15,7]` → `2` (leaf `9`) · `[2,null,3,null,4,null,5,null,6]` → `5` (only a right spine, so the lone leaf is at depth 5).
**Constraints:** `0 … 10^5` nodes (**tree can be empty** → return `0`), values `-1000 … 1000`.

> **The whole problem is the one-child trap.** For a node with only one child, the missing side is **not** a leaf at depth 0 — you must go *down the existing child*. Naively doing `1 + min(left, right)` on a one-child node returns 1 (via the null side) and is **wrong**. Either recurse only into real children, or special-case "one child missing → take the other side."

## Idea (1-liner)
Level-order BFS; the **first node you pop that has no children** sits at the minimum depth — return that depth immediately.

## Approach
1. Empty tree → return `0`.
2. Seed queue with `root`, `depth = 1`.
3. Per level: snapshot `sz = q.size()`, drain `sz` nodes.
4. For each popped node: if it's a **leaf** (`!left && !right`) → **return `depth`**. Else push its non-null children.
5. After draining the level, `++depth`. (BFS guarantees the first leaf hit is the shallowest.)

## Code (clean canonical)
```cpp
class Solution {
public:
    int minDepth(TreeNode* root) {
        if (!root) return 0;
        queue<TreeNode*> q;
        q.push(root);
        int depth = 1;

        while (!q.empty()) {
            for (int sz = q.size(); sz > 0; --sz) {     // snapshot level size
                TreeNode* n = q.front(); q.pop();
                if (!n->left && !n->right) return depth; // first leaf = answer
                if (n->left)  q.push(n->left);
                if (n->right) q.push(n->right);
            }
            ++depth;                                     // advance only after draining the level
        }
        return depth;
    }
};
```
**TC:** O(n) worst case, but **short-circuits at the first leaf** — far better on deep trees with a shallow leaf.  **SC:** O(n) — queue holds up to a full level (~n/2 leaves worst case).

## My Solution (recursive DFS with sentinels — correct)
My version recursed and used a large sentinel so the missing side loses the `min`:
```cpp
int minDepth(TreeNode* root) {
    if (!root) return 0;
    if (!root->left && !root->right) return 1;
    int left = 1e5 + 1, right = 1e5 + 1;
    if (root->left)  left  = minDepth(root->left);
    if (root->right) right = minDepth(root->right);
    return 1 + min(left, right);
}
```
Correct: it only recurses into real children, so a one-child node can't pick up the bogus null side. The sentinel `1e5 + 1` is safe only because max depth ≤ #nodes ≤ `1e5`, so it always loses the `min` — it's a magic number that leans on the constraint. The sentinel-free form below is cleaner.

## Other Approaches
- **Brute force — enumerate all root→leaf paths (DFS), take the min length.** Recurse to every leaf, track path length, keep the smallest. O(n) time but explores the *whole* tree even when a shallow leaf exists. Only worth it if you also need the path itself.
- **DFS without the sentinel (cleaner recursion), O(h) space.** When one child is null its subtree returns 0, so `1 + l + r` collapses to the existing side; otherwise take the `min`. Same O(n) time, O(h) stack — beats BFS on space for tall/balanced trees.
  ```cpp
  int minDepth(TreeNode* root) {
      if (!root) return 0;
      int l = minDepth(root->left), r = minDepth(root->right);
      return (!root->left || !root->right) ? 1 + l + r : 1 + min(l, r);
  }
  ```
- **BFS, return at first leaf (optimal / canonical interview answer).** The code above. Same O(n) worst case but **short-circuits** — the expected answer for "minimum depth" because it stops the moment it can, whereas every DFS must finish. Reach for this when the interviewer stresses "shortest" or the tree may be deep with a near-root leaf.

## Pitfalls
- **The one-child trap.** `1 + min(maxDepth-left, maxDepth-right)` is wrong when one child is null: it returns the path to a *non-leaf*. Recurse only into real children, or special-case one-missing-child. This is *the* bug of the problem.
- **Empty tree → `0`** (constraints allow 0 nodes). Don't push a null root.
- **`++depth` only after draining the level**, not per node — otherwise depth counts nodes, not levels.
- **Sentinel magic numbers** (`1e5 + 1`) work only because max depth ≤ constraint; prefer the sentinel-free DFS or BFS for clarity.
- Inherited from 102: **snapshot `q.size()` before the inner loop**, **push only non-null children**, **left-before-right**.
