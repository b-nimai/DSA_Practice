# 8. 117. Populating Next Right Pointers in Each Node II (LC 117) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/populating-next-right-pointers-in-each-node-ii/)

**Pattern:** Tree BFS — **identical task to [[7-116-populating-next-right-pointers-in-each-node|116]] but on *any* binary tree (not perfect).** The queue solution is byte-for-byte the same; the *whole point* of 117 is that the O(1) follow-up can no longer assume a child exists, so it needs a **dummy node** to discover the next level's children while scanning the current one.
**Trigger:** "**connect each node to its right neighbour** on a **general/any** binary tree", "next right pointer, ragged levels" → 102 BFS, link node `i` → `q.front()`. If **"constant extra space"**: walk the current level via `next`, threading every real child onto a `dummy → prev` chain; `dummy.next` is the next level's head.

## Problem (1-liner)
In **any** binary tree (parents may have 0, 1, or 2 children, levels are ragged), set every node's `next` to the node immediately to its **right on the same level**; the rightmost node of each level → `NULL` (already the initial value).
**Examples:** `[1,2,3,4,5,null,7]` → `[1,#,2,3,#,4,5,7,#]` (node 3 has no left child, so `5->next = 7`) · `[]` → `[]`.
**Constraints:** `0 … 6000` nodes (**tree can be empty**), values `-100 … 100`. All `next` start as `NULL`. Follow-up: **O(1) extra space** (recursion stack doesn't count).

> **The only difference from 116 is the tree shape, and it only bites the O(1) version.** In 116 (perfect) you can hardcode `head->left->next = head->right` and `head->right->next = head->next->left` — every child is guaranteed present. Here any of `head->left`, `head->right`, `head->next->left` can be null, and you don't even know where the next level starts. So you **scan** the current level via `next` and **collect** real children onto a dummy-headed chain. That dummy-node code also solves 116 — it's the strictly more general answer.

## Idea (1-liner)
Per level, link each node to the next in the queue; for O(1) space, thread the next level's existing children onto a `dummy` sentinel as you walk the current level via `next`.

## Approach (O(1) — dummy node)
1. `curr = root` (head of the current, already-linked level).
2. Per level: create a fresh `dummy`, set `prev = &dummy`.
3. Walk the current level via `next`: for each `curr`, if `left` exists append it (`prev->next = left; prev = prev->next`), same for `right`. This stitches the next level left→right.
4. `curr = dummy.next` — the head of the level we just built — and repeat. Stop when no children were collected (`dummy.next == NULL`).

## Code (clean canonical — O(1) space, also solves 116)
```cpp
class Solution {
public:
    Node* connect(Node* root) {
        Node* curr = root;
        while (curr) {
            Node dummy(0);                 // head-sentinel for the NEXT level
            Node* prev = &dummy;
            while (curr) {                 // walk current level via next
                if (curr->left)  { prev->next = curr->left;  prev = prev->next; }
                if (curr->right) { prev->next = curr->right; prev = prev->next; }
                curr = curr->next;
            }
            curr = dummy.next;             // descend to the level just stitched
        }
        return root;
    }
};
```
**TC:** O(n) — each node visited once. **SC:** O(1) — just `dummy` + `prev` per level. Meets the follow-up.

> **Why the dummy node:** without it you'd special-case "is this the first child of the next level?" on every append. The sentinel makes `prev->next = child` uniform — the first real child lands on `dummy.next` (your next `curr`), the rest chain naturally.

## My Solution (BFS with queue — correct, O(n) space)
The queue version from 116 works **unchanged** here (it never assumed perfectness):
```cpp
Node* connect(Node* root) {
    if (!root) return root;
    queue<Node*> q; q.push(root);
    while (!q.empty()) {
        int size = q.size();
        for (int i = 0; i < size; i++) {
            Node* n = q.front(); q.pop();
            if (i < size - 1) n->next = q.front();
            if (n->left)  q.push(n->left);
            if (n->right) q.push(n->right);
        }
    }
    return root;
}
```
Correct and clean, but **O(n) space** — fails the constant-space follow-up. Lead with this, then reach for the dummy node when pushed on space.

## Other Approaches
- **Brute force — BFS with a queue (above).** Same code as 116. O(n) time, **O(n) space**. The "explain-first" answer.
- **O(1) space — dummy node (optimal / canonical).** The code above. O(n) time, O(1) space. The expected answer once "constant extra space" is stated. Generalises 116's two-liner.
- **DFS recursion.** Connect children using the parent's already-set `next`, but you must recurse the **right subtree before the left** so a node's `next` chain is built before its children query it. O(n) time, O(h) implicit stack (problem says stack doesn't count). Trickier to get right than the dummy-node iteration — prefer the iterative version.

## Pitfalls
- **Don't reuse 116's O(1) code here.** `head->left->next = head->right` assumes both children exist; in 117 either can be null and `head->next->left` can be null. Use the dummy-node scan instead.
- **Fresh `dummy` per level** — reset `prev = &dummy` each outer iteration, or you'll cross-link levels.
- **Loop ends naturally** when a level produces no children (`dummy.next == NULL` ⇒ outer `while (curr)` exits). No explicit leaf check needed.
- **DFS must go right-subtree-first**, else a node's `next` isn't set when its children need it.
- Inherited from 102: in the BFS variant, **snapshot `q.size()`**, **push only non-null children**, **left-before-right**. Empty tree → return `root` (`NULL`).
