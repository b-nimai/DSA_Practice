# 10. 1161. Maximum Level Sum of a Binary Tree (LC 1161) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/maximum-level-sum-of-a-binary-tree/)

**Pattern:** Tree BFS — **102 skeleton, fold each level into one aggregate (`sum`), but return the *level index* of the best aggregate, not the aggregate.** A twist on [[5-637-average-of-levels-in-binary-tree|637 Average]] / [[9-515-find-largest-value-in-each-tree-row|515 Max-per-row]]: same per-level reduce, but you track **which** level won.
**Trigger:** "**smallest level x with maximal sum**", "best/argmax level by sum", "return the depth, not the value" → 102 BFS, sum each level, keep `bestLevel` on **strict** improvement (`<`), levels are **1-indexed**.

## Problem (1-liner)
Levels are **1-indexed** (root = level 1). Return the **smallest level `x`** whose node-sum is **maximal**.
**Examples:** `[1,7,0,7,-8,null,null]` → `2` (sums: L1=1, L2=7, L3=−1) · `[989,null,10250,98693,-89388,null,null,null,-32127]` → `2`.
**Constraints:** `1 … 10^4` nodes (**tree is never empty**), values `−10^5 … 10^5` (negatives → a level sum can be negative; sum stays within `int`, max ≈ `10^4 · 10^5 = 10^9` < `2^31`).

> **Use a strict `if (maxSum < sum)` so ties keep the *earliest* level.** The problem wants the **smallest** `x` among maximal levels. `<=` would overwrite on a tie and return the **last** max level — *the* bug here. Also seed `maxSum = INT_MIN` (not `0`): an all-negative tree must still pick a real level.

## Idea (1-liner)
Standard level-order; sum each level; remember the level number of the max sum, updating only on a strictly greater sum.

## Approach
1. Seed queue with `root` (guaranteed non-null), `level = 0`, `maxSum = INT_MIN`, `bestLevel = 0`.
2. Per level: `++level`, snapshot `sz = q.size()`, `sum = 0`, drain `sz` nodes adding `node->val`, push non-null children.
3. After draining, if `maxSum < sum` → `maxSum = sum`, `bestLevel = level`.
4. Return `bestLevel`.

## Code (clean canonical)
```cpp
class Solution {
public:
    int maxLevelSum(TreeNode* root) {
        int bestLevel = 0, maxSum = INT_MIN, level = 0;
        queue<TreeNode*> q;
        q.push(root);

        while (!q.empty()) {
            ++level;
            int sum = 0;
            for (int sz = q.size(); sz > 0; sz--) {     // snapshot level size
                TreeNode* node = q.front(); q.pop();
                sum += node->val;
                if (node->left)  q.push(node->left);
                if (node->right) q.push(node->right);
            }
            if (maxSum < sum) {                          // STRICT < → smallest level wins ties
                maxSum = sum;
                bestLevel = level;
            }
        }
        return bestLevel;
    }
};
```
**TC:** O(n) — each node visited once. **SC:** O(n) — queue holds up to a full level (~n/2 leaves).

## Other Approaches
- **BFS, per-level sum + argmax (above) — optimal / canonical.** O(n) time, O(n) space. The pattern-true answer for "best level by aggregate."
- **DFS with depth index (better space).** Recurse carrying `depth`; accumulate into `vector<long> sums` (push a slot on first reach), then scan for the max and return `index + 1`:
  ```cpp
  void dfs(TreeNode* n, int d, vector<long>& sums) {
      if (!n) return;
      if (d == (int)sums.size()) sums.push_back(0);
      sums[d] += n->val;
      dfs(n->left, d + 1, sums); dfs(n->right, d + 1, sums);
  }
  // answer: argmax over sums, +1 (1-indexed). Take the FIRST max for smallest level.
  ```
  O(n) time, **O(h) stack** + O(levels) array — beats BFS on tall/balanced trees. When scanning, use strict `>` so the **first** (smallest) max level wins.

## Pitfalls
- **Strict `<` (or `>` when scanning), never `<=`** — ties must keep the **smallest** level; `<=` returns the last max level. The signature trap of this problem.
- **Levels are 1-indexed** — increment `level` at the *top* of the loop (root = 1). DFS variant must `+1` the array index.
- **Seed `maxSum = INT_MIN`, not `0`** — an all-negative tree must still select a real level; `0` would falsely win.
- **Snapshot `q.size()` before the inner loop** — reading it in the condition after pushing children merges levels. *The* Tree BFS bug.
- Inherited from 102: **push only non-null children**, **left-before-right**. Tree is never empty (≥ 1 node), so no empty-tree guard needed.
