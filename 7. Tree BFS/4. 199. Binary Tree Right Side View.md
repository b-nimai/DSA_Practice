# 4. 199. Binary Tree Right Side View (LC 199) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/binary-tree-right-side-view/)

**Pattern:** Tree BFS — **102 skeleton, keep only the last node per level**. Same level-size-snapshot loop; instead of collecting the whole level, you record just the rightmost value.
**Trigger:** "**right side view**", "nodes you can **see from the right**", "**rightmost node of each level**", "top-to-bottom silhouette" → plain level order BFS, push the **last** node of each level. (Left side view = symmetric: push the *first* node per level.)

## Problem (1-liner)
Standing on the right of the tree, return the values visible top→bottom — i.e. the **rightmost node at each depth**.
**Examples:** `[1,2,3,null,5,null,4]` → `[1,3,4]` · `[1,2,3,4,null,null,null,5]` → `[1,3,4,5]` · `[1,null,3]` → `[1,3]` · `[]` → `[]`.
**Constraints:** `0 … 100` nodes, values `[-100, 100]`. Empty tree → `[]`.

> **Why it's the rightmost, not just the right child:** if a node has no right child, its deepest *left* descendant becomes visible (see ex. 2 & 3). So "right side view" ≠ "go right always" — it's the last node *in level order*, which BFS gives you for free.

## Idea (1-liner)
Run plain left→right BFS. For each level, the **last node** drained from the queue is the one you see from the right — record it. With `q.back()` you can grab it in O(1) *before* the inner loop even starts.

## Approach
1. Empty-tree guard → return `{}`.
2. Standard 102 BFS: queue seeded with `root`, snapshot `size = q.size()` per level.
3. **Before draining the level, read `q.back()->val`** — that's the current level's rightmost node — and push it to the result.
4. Drain all `size` nodes, enqueueing non-null children left→right.

## Code (clean canonical)
```cpp
class Solution {
public:
    vector<int> rightSideView(TreeNode* root) {
        vector<int> rightView;
        if (!root) return rightView;

        queue<TreeNode*> q;
        q.push(root);

        while (!q.empty()) {
            int size = q.size();
            rightView.push_back(q.back()->val);   // rightmost of this level, O(1)

            for (int i = 0; i < size; ++i) {
                TreeNode* node = q.front(); q.pop();
                if (node->left)  q.push(node->left);
                if (node->right) q.push(node->right);
            }
        }
        return rightView;
    }
};
```
**TC:** O(n) — BFS visits each node once.  **SC:** O(n) — queue holds up to a full level (worst case ~n/2 leaves).

## My Solution (passed all tests — refinement notes)
My original was correct: I tracked `rightMost = node->val` for *every* node and used the last write. Two refinements, both readability not asymptotics:
1. **Read the rightmost directly with `q.back()->val`** at the top of the level instead of overwriting a `rightMost` var on every iteration. The queue currently holds exactly this level's nodes (children aren't pushed yet), so `q.back()` *is* the rightmost — one read, clearer intent, no per-node assignment.
2. **Cosmetic spacing** — `vector<int> rightView;`, `queue<TreeNode*> q;`.

The overwrite-every-node version is equally O(n) and a perfectly valid answer; the `q.back()` form just states "I want the last node of the level" outright.

## Other Approaches
- **DFS, right child first (O(h) space).** Recurse carrying `depth`; the **first** node seen at each depth is the rightmost. Better space on balanced trees (stack height vs. queue width).
  ```cpp
  void dfs(TreeNode* node, int depth, vector<int>& res) {
      if (!node) return;
      if (depth == (int)res.size()) res.push_back(node->val); // first hit at this depth = rightmost
      dfs(node->right, depth + 1, res);   // right FIRST
      dfs(node->left,  depth + 1, res);
  }
  ```
  Good interview follow-up when asked to cut space on a balanced tree. Trap: must recurse **right before left**, and gate on `depth == res.size()` so only the first node per depth is kept.
- **BFS tracking `rightMost` per node (my original).** Same complexity; just less direct than `q.back()`.

## Pitfalls
- **"Right side" ≠ always-go-right.** A missing right child exposes the deepest left descendant — you must take the rightmost *in level order*, not the right pointer. `q.back()` / last-drained node handles this automatically.
- **Read `q.back()` before pushing children** — once you enqueue this level's children, `q.back()` becomes a next-level node. Read it at the top of the `while`, before the inner loop.
- **Empty-tree guard** — return `{}` on null root.
- Inherited from 102: **snapshot `q.size()` before the inner loop**, **push only non-null children**, **left-before-right** enqueue.
