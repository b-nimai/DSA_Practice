# 9. 515. Find Largest Value in Each Tree Row (LC 515) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/find-largest-value-in-each-tree-row/)

**Pattern:** Tree BFS — **102 skeleton, fold each level into one aggregate (`max`).** Literally [[5-637-average-of-levels-in-binary-tree|637 Average of Levels]] with `max` instead of `sum` — and *easier*, because comparing has no overflow/`double` trap that summing does.
**Trigger:** "**largest / max value in each row**", "one aggregate (max/min) per depth", "per-level reduce" → 102 BFS, seed `maxi = INT_MIN` per level, fold the level, push `maxi`.

## Problem (1-liner)
Return a vector with the **largest value of each row** (level), top-down, 0-indexed.
**Examples:** `[1,3,2,5,3,null,9]` → `[1,3,9]` · `[1,2,3]` → `[1,3]`.
**Constraints:** `0 … 10^4` nodes (**tree can be empty** → `[]`), values span the **full `int` range** `-2^31 … 2^31-1` (so a value can be `INT_MIN`).

> **Seed the per-level max with `INT_MIN`, not `0`.** Values can be negative (down to `INT_MIN`), so a `0` seed would wrongly report `0` for an all-negative level. `INT_MIN` loses to any real value, including `INT_MIN` itself (`max(INT_MIN, INT_MIN) = INT_MIN`). No overflow risk — you compare, never sum.

## Idea (1-liner)
Per level, snapshot the size, take the `max` of the level's values, push it to the result.

## Approach
1. Empty tree → return `[]`.
2. Seed queue with `root`.
3. Per level: snapshot `sz = q.size()`, set `maxi = INT_MIN`, drain `sz` nodes.
4. For each popped node: `maxi = max(maxi, node->val)`, push non-null children.
5. After draining, `result.push_back(maxi)`. Return `result`.

## Code (clean canonical)
```cpp
class Solution {
public:
    vector<int> largestValues(TreeNode* root) {
        vector<int> result;
        if (!root) return result;

        queue<TreeNode*> q;
        q.push(root);

        while (!q.empty()) {
            int maxi = INT_MIN;
            for (int sz = q.size(); sz > 0; sz--) {     // snapshot level size
                TreeNode* node = q.front(); q.pop();
                maxi = max(maxi, node->val);
                if (node->left)  q.push(node->left);
                if (node->right) q.push(node->right);
            }
            result.push_back(maxi);
        }
        return result;
    }
};
```
**TC:** O(n) — each node visited once. **SC:** O(n) — queue holds up to a full level (~n/2 leaves); output O(height) isn't counted as extra.

## Other Approaches
- **BFS, per-level max (above) — optimal / canonical.** O(n) time, O(n) space. The pattern-true answer for "aggregate per level."
- **DFS with depth index (better space).** Recurse carrying `depth`; the first time you reach a new depth push the value, otherwise `res[depth] = max(res[depth], node->val)`:
  ```cpp
  void dfs(TreeNode* n, int d, vector<int>& res) {
      if (!n) return;
      if (d == (int)res.size()) res.push_back(n->val);
      else res[d] = max(res[d], n->val);
      dfs(n->left, d + 1, res);
      dfs(n->right, d + 1, res);
  }
  ```
  O(n) time, **O(h) stack** — beats BFS on tall/balanced trees. Reach for it if space is constrained or you prefer recursion.

## Pitfalls
- **Seed `INT_MIN`, never `0`** — values can be negative; `0` corrupts all-negative levels. (Mirror of 637's "sum into `double`/`long long`" trap — both come from the full `int` value range.)
- **Snapshot `q.size()` before the inner loop** — reading it in the condition after pushing children merges levels. *The* Tree BFS bug.
- **Empty tree → `[]`** (constraints allow 0 nodes); don't push a null root.
- Inherited from 102: **push only non-null children**, **left-before-right**.
