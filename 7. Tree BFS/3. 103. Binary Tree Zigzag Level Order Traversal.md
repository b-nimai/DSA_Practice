# 3. 103. Binary Tree Zigzag Level Order Traversal (LC 103) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/binary-tree-zigzag-level-order-traversal/)

**Pattern:** Tree BFS — **102 + flip the placement of alternate levels**. Same level-size-snapshot skeleton; the *only* change is even levels go left→right, odd levels right→left (boustrophedon / snake order).
**Trigger:** "**zigzag** level order", "**snake** / spiral by level", "left→right then **right→left** alternating", "serpentine traversal" → do plain level order, but reverse the *direction* you write into every other level.

## Problem (1-liner)
Return values grouped by level, but **alternate the direction each level**: level 0 left→right, level 1 right→left, level 2 left→right, …
**Examples:** `[3,9,20,null,null,15,7]` → `[[3],[20,9],[15,7]]` · `[1]` → `[[1]]` · `[]` → `[]`.
**Constraints:** `0 … 2000` nodes, values `[-100, 100]`. Empty tree → `[]`.

## Idea (1-liner)
Traverse the tree in **plain left→right BFS** (so children-enqueue order stays correct), but **pre-size each level vector and write each value into the right slot**: `index = leftFirst ? i : sz-1-i`. A `bool` flips after every level. No `reverse()` call needed — placement *is* the flip.

## Approach
1. Run the standard 102 BFS: queue, snapshot `sz = q.size()` before the inner loop, push non-null children.
2. Pre-size `vector<int> level(sz)` and keep a `bool leftFirst` (starts `true`).
3. For the `i`-th node drained, compute `index = leftFirst ? i : sz-1-i` and write `level[index] = node->val`.
4. After the level, `leftFirst = !leftFirst` and `result.push_back(level)`.

> **Why the index trick beats `reverse`:** the BFS *enqueue* order must stay left→right or the next level's children come out mirrored. So you can't reverse the traversal — you reverse the **write position**. This puts each value directly in its final slot, no second pass.

## Code (clean canonical)
```cpp
class Solution {
public:
    vector<vector<int>> zigzagLevelOrder(TreeNode* root) {
        vector<vector<int>> result;
        if (!root) return result;

        queue<TreeNode*> q;
        q.push(root);
        bool leftFirst = true;

        while (!q.empty()) {
            int sz = q.size();              // snapshot the level boundary
            vector<int> level(sz);          // pre-sized so we can index any slot

            for (int i = 0; i < sz; ++i) {
                TreeNode* node = q.front(); q.pop();
                int index = leftFirst ? i : sz - 1 - i;   // flip write position
                level[index] = node->val;
                if (node->left)  q.push(node->left);
                if (node->right) q.push(node->right);
            }
            result.push_back(level);
            leftFirst = !leftFirst;         // alternate direction each level
        }
        return result;
    }
};
```
**TC:** O(n) — BFS visits each node once; indexed write is O(1) (no reverse pass).  **SC:** O(n) — queue + output.

## Other Approaches
- **Plain BFS + `reverse(level...)` on alternate levels** — also O(n) overall, but does an extra pass per odd level and reads less intentionally. The index trick writes each value into its final slot in one pass — strictly tidier. *Avoid as the default.*
- **`deque` per level (push_front on right→left levels, push_back on left→right)** — single pass, O(1) inserts, no reverse, but adds branching inside the hot loop for no asymptotic gain vs. the index math.
- **DFS with depth** — recurse carrying `depth`, append to `result[depth]`, then reverse the odd-indexed levels at the end. Same result, different traversal; loses BFS's natural level boundaries.

## Pitfalls
- **Don't reverse the traversal/enqueue order to get zigzag** — that mirrors the *next* level's children. Always BFS left→right; flip only the **write index** (or reverse the finished level vector). This is *the* 103 trap.
- **Pre-size the level vector (`vector<int> level(sz)`)** before indexing — writing `level[sz-1-i]` into an empty/`push_back`-grown vector is UB.
- **Flip `leftFirst` once per level, after the inner loop** — not per node.
- Everything inherited from 102 still applies: **snapshot `q.size()` before the inner loop**, **empty-tree guard**, **push only non-null children**, **left-before-right** enqueue.
