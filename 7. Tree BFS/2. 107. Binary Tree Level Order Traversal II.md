# 2. 107. Binary Tree Level Order Traversal II (LC 107) — Medium
🔗 [Open on LeetCode](https://leetcode.com/problems/binary-tree-level-order-traversal-ii/)

**Pattern:** Tree BFS — **102 + one final `reverse`**. Same level-size-snapshot skeleton; the *only* change is the output is bottom-up (leaf level first, root level last).
**Trigger:** "**bottom-up** level order", "level by level **from leaf to root**", "levels in reverse" → do plain level order, then reverse the outer list of levels.

## Problem (1-liner)
Return values grouped by level, but **bottom level first**: leaves' level → … → root's level.
**Examples:** `[3,9,20,null,null,15,7]` → `[[15,7],[9,20],[3]]` · `[1]` → `[[1]]` · `[]` → `[]`.
**Constraints:** `0 … 2000` nodes, values `[-1000, 1000]`. Empty tree → `[]`.

## Idea (1-liner)
Bottom-up is just top-down **flipped**. Build the levels exactly as in [[1-102-binary-tree-level-order-traversal|102]], then `reverse` the **outer** vector once at the end — that swaps the order of the levels without touching the values inside each level.

## Approach
1. Run the standard 102 BFS: queue, snapshot `sz = q.size()` before the inner loop, drain `sz` nodes into a per-level vector, push non-null children.
2. After the loop, `reverse(result.begin(), result.end())`.
3. Return.

> The reverse is **O(#levels)**, not O(n): a `vector<vector<int>>` reverse only swaps the *outer* elements (each is effectively a pointer/handle to a level's buffer). The ints inside are never moved or copied.

## Code (clean canonical)
```cpp
class Solution {
public:
    vector<vector<int>> levelOrderBottom(TreeNode* root) {
        vector<vector<int>> result;
        if (!root) return result;

        queue<TreeNode*> q;
        q.push(root);

        while (!q.empty()) {
            int sz = q.size();             // snapshot the level boundary
            vector<int> level;
            level.reserve(sz);

            for (int i = 0; i < sz; ++i) {
                TreeNode* node = q.front(); q.pop();
                level.push_back(node->val);
                if (node->left)  q.push(node->left);
                if (node->right) q.push(node->right);
            }
            result.push_back(level);
        }

        reverse(result.begin(), result.end());   // top-down → bottom-up
        return result;
    }
};
```
**TC:** O(n) — BFS visits each node once; the final reverse is O(#levels) ≤ O(n).  **SC:** O(n) — queue + output.

## Other Approaches
- **Front-insert each level (`result.insert(result.begin(), level)`)** — *avoid.* Inserting at the front of a `vector` is **O(n)** (shifts everything right), so once per level makes the whole solution **O(levels²)**. The single end-`reverse` is strictly better.
- **Build into a `deque` with `push_front`** — O(1) front insertion, no reverse needed, but you'd return a `deque` (or copy it back to a vector), which is more awkward than just reversing. Only worth it if a `deque` return is acceptable.
- **DFS with depth** — recurse carrying `depth`, fill `result[depth]`, then reverse at the end. Same idea, different traversal.

## Pitfalls
- **Reverse once at the end, don't front-insert per level** — front-insertion on a vector is O(n) → O(levels²) overall. The classic efficiency trap here.
- **Reverse the *outer* vector, not the inner ones** — each level's left-to-right order must stay intact (`[15,7]`, not `[7,15]`); only the order *of the levels* flips.
- Everything inherited from 102 still applies: **snapshot `q.size()` before the inner loop**, **empty-tree guard**, **push only non-null children**, **left-before-right**.
