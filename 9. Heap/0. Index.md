# Heap — Index

60-second whole-pattern recall. Read the **Trigger** column and try to recall the **Idea** before peeking.

**Core skeleton (Top-K):** to keep the **k largest**, maintain a **min-heap of size k** — push each element, pop when size exceeds k; the top is the kth largest. Flip to a **max-heap of size k** for the k smallest. Heap ops are O(log k), space O(k). For a static array, **quickselect / `nth_element`** beats the heap at average O(n).

## When to reach for Heap
1. **"Top K / K largest / K closest / K most frequent"** → size-K heap (min-heap for largest, max-heap for smallest/closest), O(n log k).
2. **Running median / balance two halves of a stream** → two heaps (max-heap low half + min-heap high half).
3. **Repeatedly pull the current min/max** (Dijkstra, merge K lists, scheduling) → priority queue as the frontier.
4. **One-shot kth order statistic on a static array** → quickselect / `nth_element`, average O(n).

## Common Pitfalls Across the Pattern
- **Wrong heap polarity** — kth *largest* → **min**-heap of size k; kth *smallest* → **max**-heap of size k. Getting this backwards is the #1 bug.
- **Letting the heap grow to n** — cap it at k for O(k) space and O(log k) ops.
- **`priority_queue` default is a max-heap** — pass `greater<T>` (and the `vector<T>` middle arg) to get a min-heap.
- **Unsigned `size()`** — comparing `heap.size() > k` with `int k` warns/surprises; cast or use `size_t`.
