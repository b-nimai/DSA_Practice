# Heap вЂ” Index

60-second whole-pattern recall. Read the **Trigger** column and try to recall the **Idea** before peeking.

**Core skeleton (Top-K):** to keep the **k largest**, maintain a **min-heap of size k** вЂ” push each element, pop when size exceeds k; the top is the kth largest. Flip to a **max-heap of size k** for the k smallest. Heap ops are O(log k), space O(k). For a static array, **quickselect / `nth_element`** beats the heap at average O(n).

## When to reach for Heap
1. **"Top K / K largest / K closest / K most frequent"** в†’ size-K heap (min-heap for largest, max-heap for smallest/closest), O(n log k).
2. **Running median / balance two halves of a stream** в†’ two heaps (max-heap low half + min-heap high half).
3. **Repeatedly pull the current min/max** (Dijkstra, merge K lists, scheduling) в†’ priority queue as the frontier.
4. **One-shot kth order statistic on a static array** в†’ quickselect / `nth_element`, average O(n).

## Common Pitfalls Across the Pattern
- **Wrong heap polarity** вЂ” kth *largest* в†’ **min**-heap of size k; kth *smallest* в†’ **max**-heap of size k. Getting this backwards is the #1 bug.
- **Letting the heap grow to n** вЂ” cap it at k for O(k) space and O(log k) ops.
- **`priority_queue` default is a max-heap** вЂ” pass `greater<T>` (and the `vector<T>` middle arg) to get a min-heap.
- **Unsigned `size()`** вЂ” comparing `heap.size() > k` with `int k` warns/surprises; cast or use `size_t`.
